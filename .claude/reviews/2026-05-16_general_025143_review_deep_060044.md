## Concerns

1. **Misleading `--bare` comment in `lib-common.sh:claude_p_with_timeout`**. The function comment reads:
   > `--bare skips hooks/plugins/LSP for sub-sessions: faster, and prevents recursive hook triggers if tags appear in the embedded log content.`

   But `--bare` is NOT in the actual `claude -p` invocation. The command is `timeout ... claude -p "$@" < "$tmpfile"` with no `--bare`. The comment describes it as if it's an active protection mechanism when it isn't — the recursion guard is the file lock, not `--bare`. A future developer reading this will be confused about why recursion is still prevented without `--bare`, or may remove the lock thinking `--bare` covers it.

2. **`sanitize_tags` regex uses `*` for docfresh slug while tag detection uses `+`**. In `extract-turn.sh:30`:
   ```bash
   sed -E 's/\[(initprompt|phaseprompt|wrapup|review|deepreview|docfresh:[a-z0-9-]*)\]/~\1~/g'
   ```
   The `*` allows an empty slug (`[docfresh:]`). `WORKFLOW_TAGS_DOCFRESH` in `lib-common.sh:28` uses `+` (requires at least one char). The sanitizer will escape `[docfresh:]` which the detector would never recognize as a real tag, so this is harmless — but it's an inconsistency between the two patterns that could mask a future bug.

3. **Lock is acquired but `clear_turn_state` is called before lock release when the second case hits `*`**. When `TAG_TYPE` is `none`, `init`, or `phase`, the first `case` block doesn't match (no lock acquired), so the second case hits `*` (no-op). `clear_turn_state` is called at line 157, then `exit 0`. This is correct — but the structure of two sequential `case` statements creates a subtle coupling: if a new tag type is ever added to the first case without a matching branch in the second, the lock will be acquired and never released (since the `*` branch in the second case doesn't call `rm -f "$WORKFLOW_LOCK"`). The current code is correct; this is a fragility concern for future modifications.

4. **`#ASSISTANT` text is sanitized but tool call `description` fields are not**. `sanitize_tags` is applied to the `#PROMPT` section (line 77) and `#ASSISTANT` text blocks (line 131). Bash tool call `description` fields and other tool input fields are logged without sanitization. If a Bash description contained `[wrapup]` it would appear unsanitized in the log. Low probability in practice but the coverage is incomplete.

5. **`#TURN_END` marker is not written in the empty-transcript-slice early-exit path** (`extract-turn.sh:52-56`). When `TRANSCRIPT_SLICE` is empty, the script writes a `#PROMPT` block and `exit 0` without appending `#TURN_END`. This makes log format inconsistent across turns. The log shows `## #TURN_END` appearing normally, so the common path works — this is a pre-existing gap in the degenerate case, not introduced by these changes.
