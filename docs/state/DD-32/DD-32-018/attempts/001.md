## Why This Attempt Succeeded

The root cause was a one-liner: `const duration = toast.duration ?? 5000`. This applied a 5000ms auto-dismiss timeout to all toast variants including error, with no variant-aware default. The fix adds a variant check: `const defaultDuration = toast.variant === 'error' ? 0 : 5000`, then falls back to `defaultDuration` if no explicit duration was given. The existing `if (duration === 0) return` guard already handled the "persist" case correctly — it just needed the error variant to reach it by default.
