## Why This Attempt Succeeded

The root cause was that `<ToastPrimitive.Root>` uses Radix UI's built-in duration prop (defaulting to 5000ms) to schedule an internal timer that calls `onOpenChange(false)` after timeout. Our `onOpenChange` handler calls `dismiss(toast.id)`, so error toasts were being auto-dismissed by Radix's timer even though our custom `useEffect` correctly skipped scheduling its own timer. Passing `duration={Infinity}` for error toasts disables Radix's internal timer entirely, leaving only the manual dismiss path via the X button.
