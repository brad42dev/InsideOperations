## What Was Attempted

Added toast notifications to the Console workspace creation flow in `frontend/src/pages/console/index.tsx`.

The fix uses a `pendingCreateIdsRef` (a `Set<string>` held in a `useRef`) to distinguish new workspace creations from routine auto-saves, both of which go through the same `saveMutation`. When `createWorkspace()` is called with `useApi=true`, the new workspace's ID is added to the set before `persistWorkspace()` fires. In `saveMutation.onSuccess`, if the saved workspace ID is in the set, a "Workspace created" success toast is shown. In `saveMutation.onError`, if the ID is in the set, a persistent (`duration: 0`) error toast is shown with the failure reason instead of entering the auto-save retry loop.
