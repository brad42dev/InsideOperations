## What Was Attempted

Added SHA-256 checksum integrity support to the `.iographic` export/import pipeline in `services/api-gateway/src/handlers/iographic.rs`.

Three specific changes:

1. Added `#[serde(default)]` to the `checksum` field in `IographicManifest` struct so that packages lacking a `checksum` field (pre-feature packages) can still be deserialized without a hard failure.

2. In `analyze_iographic`: wrapped the existing checksum verification block in an `is_empty()` guard — if checksum is absent, log a `tracing::warn!` and skip verification; if checksum is present and mismatches, push `"Package integrity check failed"` into the errors vec (replacing the previous verbose "Checksum mismatch: expected..." message).

3. In `commit_iographic`: added checksum verification after manifest parsing. Changed `std::io::Cursor::new(bytes)` to `std::io::Cursor::new(bytes.clone())` so `bytes` stays in scope for reuse. Added the full verify block with `"Package integrity check failed"` as a `BadRequest` rejection on mismatch.
