## What Was Attempted

Added .iographic export capability to the Dashboard Builder editor at `/designer/dashboards/:id/edit`.

The root cause was that the DashboardBuilder component (used for the dashboard designer route) had no export option — its toolbar showed only Variables, Published toggle, Cancel, Save. The main Designer page (for graphics) already had a File menu with Export .iographic, but the DashboardBuilder is a completely separate component in the dashboards module.

Changes made:
1. **Backend**: Added `export_dashboard_iographic` handler to `services/api-gateway/src/handlers/dashboards.rs`. The handler queries the `dashboards` table, builds a proper `.iographic` ZIP file with `manifest.json` and `dashboards/{slug}/dashboard.json`, computes a SHA-256 checksum, and returns it as a download response.
2. **Backend route**: Registered `POST /api/dashboards/:id/export/iographic` in `services/api-gateway/src/main.rs`.
3. **Frontend API**: Added `exportIographic(id, description?)` method to `dashboardsApi` in `frontend/src/api/dashboards.ts` — uses raw `fetch` to return a Blob (same pattern as `graphicsApi.exportIographic`).
4. **Frontend UI**: Added `showExportDialog` state and an Export button to the DashboardBuilder toolbar (visible only when `!isNew`). Added inline `DashboardExportDialog` component with description input, error handling, and blob-URL download trigger.
