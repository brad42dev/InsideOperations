## What Was Attempted

Fixed the React Query contract violation where the `["graphic", <uuid>]` query key's `queryFn` returned `undefined` instead of `null` or a value.

Root cause: `graphicsApi.get` in `frontend/src/api/graphics.ts` was typed as `api.get<{ data: { id, name, scene_data, ... } }>`. The API client (`client.ts`) already unwraps the outer `{ success, data }` envelope — it sets `result.data = envelope.data`. So at runtime `result.data` IS already the graphic object `{ id, name, scene_data, ... }`. The extra `{ data: ... }` wrapper in the type annotation was incorrect, making TypeScript think `result.data` had type `{ data: { ... } }` when it actually had type `{ id, name, scene_data, ... }`. As a result, `result.data.data` was always `undefined` at runtime.

Fix applied:
1. Corrected the type annotation for `graphicsApi.get` in `graphics.ts` — removed the phantom outer `{ data: }` wrapper from the generic type parameter.
2. Updated the `queryFn` in `process/index.tsx` to use `result.data ?? null` instead of `result.data.data`.
3. Updated all other callers of `graphicsApi.get` that used `result.data.data` (or `resp.data.data`) to use `result.data` instead — in `console/panes/GraphicPane.tsx`, `forensics/EvidenceRenderer.tsx`, and `designer/index.tsx`.
