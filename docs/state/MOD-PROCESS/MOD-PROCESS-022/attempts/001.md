## What Was Attempted

Identified that the AppShell header (`<header>` element in `frontend/src/shared/layout/AppShell.tsx`) had an incorrect hide condition. The condition `(!isKiosk && topbarHidden) ? 0 : var(--io-topbar-height)` only hid the header when both NOT in kiosk mode AND topbar was manually hidden. When kiosk mode was activated via `/process?kiosk=true` (which calls `setKiosk(true)` from the Process module's own useEffect but does NOT call `setTopbarHidden(true)`), the condition evaluated to false and the header remained visible at full height.

Fixed by changing the condition to `(isKiosk || topbarHidden)` so the header is hidden when either kiosk mode is active OR the user has manually hidden the topbar.

## Files Modified
