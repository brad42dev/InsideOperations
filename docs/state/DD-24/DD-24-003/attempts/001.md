## What Was Attempted

Added RBAC permission enforcement to the `/api/import/*` proxy handler in the API Gateway, and added frontend permission guards to the Import settings page.

**Backend:**
- Modified `proxy_import` in `services/api-gateway/src/main.rs` to accept `Extension(claims): Extension<Arc<Claims>>` and inspect `claims.permissions` before proxying
- Added `io_auth::Claims` and `Extension`/`Json`/`IntoResponse` imports at the top of main.rs
- Permission logic: `/connector-templates` and `/connections` paths require `system:import_connections`; `/definitions` paths (without `/runs`) require `system:import_definitions`; any path containing `/runs` requires `system:import_history` (GET) or `system:import_execute` (non-GET methods)
- Returns 403 with `{"error": {"code": "FORBIDDEN", "message": "Insufficient permissions"}}` when the required permission is absent

**Frontend (`frontend/src/pages/settings/Import.tsx`):**
- Added `import { usePermission } from '../../shared/hooks/usePermission'`
- `ConnectionsTab`: added `canManageConnections = usePermission('system:import_connections')` — hides "New Connection" button and per-connection "Test", "Edit", "Delete" buttons
- `DefinitionsTab`: added `canManageDefinitions` and `canExecute` — hides "New Definition" button, per-definition "Dry Run"/"Run Now" (guarded by `canExecute`), and "Disable"/"Delete" (guarded by `canManageDefinitions`)
- `RunsTab`: added `canViewHistory` and `canExecute` — renders a permission-denied message if `!canViewHistory`; hides the quick-run panel and "Cancel" button from users without `canExecute`
- `ImportSettingsPage` (main component): added all four permission flags; tabs array now has `visible` field; renders only `visibleTabs` so Connectors/Connections are hidden without `system:import_connections`, Definitions hidden without `system:import_definitions`, Run History hidden without `system:import_history`
