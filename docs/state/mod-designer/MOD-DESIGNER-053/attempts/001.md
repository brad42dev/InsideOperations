## What Was Attempted

Investigated the root cause of why `#io-canvas-drag-ghost` never appeared in the DOM during UAT palette→canvas drag. Used Playwright live testing to confirm:

1. When `MouseEvent('mousedown')` is dispatched programmatically, `onMouseDown` fires and ghost IS created — works correctly.
2. When `PointerEvent('pointerdown')` is dispatched programmatically (as Playwright's CDP mouse API does), `onMouseDown` does NOT fire — ghost is never created. This was the UAT failure path.

The fix: converted all 6 palette tile drag handlers from `onMouseDown`/`React.MouseEvent` to `onPointerDown`/`React.PointerEvent`, and added `pointermove`/`pointerup` listeners alongside `mousemove`/`mouseup` for ghost tracking. Also updated Escape key cleanup handlers to remove `pointermove`/`pointerup` listeners.

## Why This Attempt Succeeded

Root cause was a pointer-event vs mouse-event mismatch. The existing code used `onMouseDown` which requires `MouseEvent` to fire. When Playwright/CDP dispatches synthetic drag events, it fires `PointerEvent('pointerdown')` without the companion `MouseEvent('mousedown')` that only real browser input generates. Switching to `onPointerDown` means the handler fires regardless of which event type the test infrastructure uses. Adding `pointermove`/`pointerup` listeners ensures the ghost position tracking and cleanup also work for pointer-only drag sequences.
