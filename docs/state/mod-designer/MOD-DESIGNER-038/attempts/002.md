## What Was Attempted

Fixed the structural rate-limiting bug in services/api-gateway/src/mw.rs where the rate_limit middleware runs before jwt_auth in the tower stack. Because Claims are not yet injected when rate_limit executes, all authenticated API requests (including /api/v1/design-objects) fall into the unauthenticated branch and are subject to the low 30 req/min per-IP limit. Replaced the narrow is_static_config_endpoint() function (which only exempted /api/notifications/channels/enabled) with is_authenticated_api_endpoint() that exempts all /api/* routes except /api/auth/*. Auth endpoints retain the tight 10/min per-IP limit for credential-stuffing protection. Also verified that the drag ghost (id="io-canvas-drag-ghost") from MOD-DESIGNER-031 is correctly implemented in DesignerCanvas.tsx with opacity:0.7, border:2px dashed, and pointer-events:none.
