## What Was Attempted

Added all missing alert API routes to the alert-service. The work covered:

1. **`main.rs`**: Added `POST /alerts` as the canonical trigger path (keeping `/alerts/trigger` as alias), `GET /alerts/active`, `GET /alerts/stats`, `GET /alerts/:id/deliveries`, `GET /alerts/:id/escalations`. Moved Twilio webhook routes out of the service-secret-protected router into a separate `webhook_routes` router so that Twilio's external HTTP calls are not blocked by the internal secret middleware.

2. **`handlers/alerts.rs`**: Added four new handler functions: `list_active_alerts` (filters `status = 'active'`), `get_stats` (severity counts, avg response time, channel reliability), `list_deliveries` (queries `alert_deliveries` by alert_id), `list_escalations` (queries `alert_escalations` by alert_id). Also added new types `AlertStats`, `SeverityCounts`, `ChannelReliability`, `AlertEscalation` — with `AlertEscalation` matching the actual schema (`from_level`, `to_level`, `reason`, `escalated_at`).

3. **`handlers/webhooks.rs`** (new file): Implemented `twilio_status` and `twilio_voice` handlers with full HMAC-SHA1 signature validation using the `hmac` + `sha1` crates. The status handler updates `alert_deliveries.status` by `external_id = MessageSid`. The voice handler acknowledges the alert when `Digits == "1"`. Auth token is fetched from the `alert_channels` table's SMS config.

4. **`handlers/mod.rs`**: Added `pub mod webhooks`.

5. **`config.rs`**: Added `twilio_status_callback_url` and `twilio_voice_callback_url` optional fields (loaded from env vars `TWILIO_STATUS_CALLBACK_URL` and `TWILIO_VOICE_CALLBACK_URL`); both default to empty/None which disables signature validation in dev mode.

6. **`Cargo.toml` (workspace)**: Added `sha1 = "0.10"` workspace dep (already in lock file as transitive dep of web-push-native).

7. **`services/alert-service/Cargo.toml`**: Added `sha1`, `hmac`, `urlencoding` dependencies.
