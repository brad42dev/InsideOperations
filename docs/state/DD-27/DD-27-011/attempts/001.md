## What Was Attempted

Investigated why /settings/sms-providers showed "Access Denied" for admin users. Traced the issue through the permission guard chain:

1. The frontend route in `frontend/src/App.tsx` uses `PermissionGuard permission="system:configure"` for the sms-providers route — this is correct.
2. The `PermissionGuard` component checks `user?.permissions.includes(permission)` — correct implementation.
3. Permissions come from the JWT token, which is populated from the database.
4. The database was missing the `system:configure` permission assignment to the Admin role because two migrations were PENDING: `20260323000001_add_missing_system_permissions` and `20260324000002_ensure_admin_system_configure`.

The fix: ran `sqlx migrate run` to apply the pending migrations, which:
- Inserted `system:configure` and `system:certificates` into the permissions table
- Assigned both permissions to the Admin role via role_permissions

## Files Modified

None — the migration SQL files already existed and correctly implemented the fix. The issue was that they had not been applied to the development database. Applied via `DATABASE_URL="postgresql://io:io_password@localhost:5432/io_dev" sqlx migrate run`.
