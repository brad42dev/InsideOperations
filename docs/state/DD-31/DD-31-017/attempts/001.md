## Why This Attempt Succeeded

The fix was straightforward: the `list_templates` handler was wrapping the templates array in a `PagedResponse` struct which the frontend's `client.ts` was detecting (via the `pagination` key) and re-wrapping, causing the frontend to call `.find()` on a paginated object rather than a plain array. By switching to the local `ok()` helper (which returns `{ "success": true, "data": [...] }`), the response is a plain array that the frontend can directly call `.find()` on.
