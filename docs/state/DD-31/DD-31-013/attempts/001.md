## What Was Attempted

Fixed `HistoryPanel` in `frontend/src/pages/alerts/index.tsx` (lines 960-961).

Root cause: The `/api/notifications/messages` backend endpoint returns a paginated response `{ success: true, data: [...], pagination: {...} }`. The API client in `client.ts` detects the `pagination` field and wraps the payload as a `PaginatedResult`: `{ data: NotificationMessage[], pagination: {...} }`. This wrapped object is stored as `result.data`.

The old code on line 960 did:
```
const messages: NotificationMessage[] = (result?.success && result.data) ? result.data : []
```

Since `result.data` is the PaginatedResult object (truthy), the conditional passed and `messages` was assigned the whole object â€” not an array. Calling `messages.map(...)` then crashed with "messages.map is not a function".

Line 961 also incorrectly tried to find `pagination` on `result` instead of `result.data`.

Fix: introduced a `HistoryData` type alias, extracted `pagedData` from `result.data`, then extracted `messages` from `pagedData.data` (with `Array.isArray` guard) and `pagination` from `pagedData.pagination`.
