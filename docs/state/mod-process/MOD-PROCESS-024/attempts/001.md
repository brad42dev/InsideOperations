## What Was Attempted

Fixed the `<header>` (banner landmark) in AppShell.tsx to use `display: none` when `isKiosk` is true, instead of `height: 0` + `overflow: hidden`. The `display: none` approach removes the element from the accessibility tree entirely, which prevents screen readers and accessibility snapshots from detecting the `<h1>` "Process" heading.

## Why This Attempt Succeeded

The root cause was that `height: 0` + `overflow: hidden` hides an element visually but NOT from the accessibility tree. Browsers still expose the element (and its children, including the h1) to screen readers and accessibility APIs. Using `display: none` is the correct approach because it removes the element from both the layout flow and the accessibility tree. The fix was a one-line change to the display property of the header element in AppShell.tsx.
