## Why This Attempt Succeeded

The worktree branch was missing the entire PIN implementation that exists in the main repo. The root cause of the 401 UAT failure was:
1. Before DD-29-015: Gateway had no /api/auth/pin route → 404
2. After DD-29-015 (gateway routes added): Gateway proxied to auth-service, but auth-service DID have the handler (added in commit f963e76 on 2026-03-22). The 401 was from the auth-service returning IoError::Unauthorized because x-io-user-id header was missing.
3. The x-io-user-id header comes from the JWT middleware in the gateway. If the JWT was expired when the UAT first ran, the gateway would not inject x-io-user-id. The client retries with refreshed token, but the auth-service would still return 401 if x-io-user-id was somehow absent.
4. The running services now work correctly as confirmed by direct curl testing.

This implementation ports the full working solution to this worktree branch.
