## What Was Attempted

Implemented the EULA hard gate across all local, OIDC, and SAML login flows in the auth-service. Changes span 5 files:

1. **state.rs** — Added `EulaPendingEntry` struct (user_id, expires_at, used) and `eula_pending_tokens: Arc<DashMap<String, EulaPendingEntry>>` field to `AppState::new()`.

2. **handlers/auth.rs** — Extended the user query to fetch `is_emergency_account`. Inserted a new step 6 (EULA gate) between the MFA gate and JWT issuance. The gate: queries whether the user has an active end_user EULA acceptance; if not (and user is not service/emergency account), generates a 32-byte hex pending token, stores it in `state.eula_pending_tokens` with 5-minute TTL, and returns `{"status": "eula_required", "eula_pending_token": "...", "eula": {...}}` without setting any cookie. Removed the now-redundant post-token EULA check (old step 10). Added `EulaRequiredResponse` and `EulaInfo` response types.

3. **handlers/eula.rs** — Added `AcceptEulaPendingBody` request type and `accept_eula_pending` handler. Handler: validates pending token from DashMap (missing → 401, used → 401, expired → 401), marks used, removes token, records acceptance in `eula_acceptances` table (same chained hash logic as existing `accept_eula`), mirrors to `users.eula_accepted`, then issues JWT access token + refresh cookie (same pattern as `mfa_verify_login`). Added necessary imports: `header`, `StatusCode`, `io_auth::{build_claims, generate_access_token}`.

4. **handlers/oidc.rs** — Added EULA gate in `oidc_callback_inner` before JWT issuance. Fetches `is_service_account` and `is_emergency_account` from users table. If EULA not accepted: issues pending token, redirects to `/eula-required?pending_token=...` (FOUND 302).

5. **handlers/saml.rs** — Added EULA gate in `issue_saml_jwt` before JWT issuance. Same pattern as oidc.rs.

6. **main.rs** — Registered `/auth/eula/accept-pending` route (before `/auth/eula/accept` to avoid path conflict). Added background tokio::spawn task sweeping `eula_pending_tokens` every 5 minutes.
