## What Was Attempted

Created a new `pin.rs` handler module in `services/auth-service/src/handlers/` implementing
three endpoints:
- `set_pin` ‚Äî POST /auth/pin: validates 6-digit numeric PIN, verifies current_password for
  local accounts (skips check for SSO-only), hashes PIN with Argon2, stores in
  `users.lock_pin_hash`.
- `delete_pin` ‚Äî DELETE /auth/pin: same current_password gate, sets `users.lock_pin_hash = NULL`.
- `verify_pin` ‚Äî POST /auth/verify-pin: checks `lock_pin_hash`, returns 404 if no PIN set,
  uses the same `user_sessions` rate-limit counters
  (`unlock_fail_count_soft`, `unlock_fail_count_hard`, `unlock_fail_window_start`) as
  `verify_password_unlock`. On success, clears `locked_since` and resets all counters.

Also added `has_pin: bool` to `MeDetail` struct in `users.rs` and populated it in `get_me`.
Registered routes in `main.rs`. Created migration for `lock_pin_hash TEXT NULL`.
