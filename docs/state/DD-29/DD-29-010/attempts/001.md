## What Was Attempted

Implemented server-side session lock persistence and WebSocket events for lock/unlock:

1. **io-bus crate** (`crates/io-bus/src/lib.rs`): Added `SessionLocked { session_id: Uuid }` and `SessionUnlocked { session_id: Uuid }` variants to `WsServerMessage` enum.

2. **auth-service config** (`services/auth-service/src/config.rs`): Added `data_broker_url` field (defaults to `http://127.0.0.1:3001`).

3. **auth-service handlers** (`services/auth-service/src/handlers/auth.rs`):
   - Added `publish_session_event` helper function that POSTs to data-broker's `/internal/publish` endpoint with service secret.
   - Added `lock_session` handler for `POST /auth/lock` — finds the most-recent active session, sets `locked_since = COALESCE(locked_since, NOW())` (idempotent), publishes `session.locked` event, writes audit log.
   - Updated `verify_password_unlock` to call `publish_session_event` with `session.unlocked` after successfully clearing `locked_since`.

4. **auth-service routes** (`services/auth-service/src/main.rs`): Registered `.route("/auth/lock", post(handlers::auth::lock_session))`.

5. **data-broker state** (`services/data-broker/src/state.rs`): Added `user_connections: Arc<DashMap<Uuid, HashSet<ClientId>>>` reverse mapping of user_id → connected client IDs.

6. **data-broker main** (`services/data-broker/src/main.rs`): Initialise `user_connections` map, pass to AppState, register `POST /internal/publish` route.

7. **data-broker ws.rs** (`services/data-broker/src/ws.rs`): On client connect, register user_id → client_id in `user_connections`. On disconnect, remove the entry.

8. **data-broker publish.rs** (`services/data-broker/src/publish.rs`): New module implementing `POST /internal/publish`. Validates `x-io-service-secret`, builds typed `WsServerMessage`, fans out to all client senders for the target `user_id`. Handles `session.locked` and `session.unlocked` event types.

9. **frontend useWsWorker.ts** (`frontend/src/shared/hooks/useWsWorker.ts`): Added `session_locked` and `session_unlocked` message handlers — both dynamically import the ui store and call `lock()` / `unlock()`. Added `sessionLockHandlers` and `sessionUnlockHandlers` sets, plus `onSessionLock` and `onSessionUnlock` public API methods.

10. **frontend auth.ts** (`frontend/src/api/auth.ts`): Added `lockSession()` API method calling `POST /api/auth/lock`.

11. **frontend AppShell.tsx** (`frontend/src/shared/layout/AppShell.tsx`): Updated idle timer callback to skip kiosk sessions (kiosk never auto-locks), added `authApi.lockSession()` call so the lock is persisted server-side. Added `authApi` import.
