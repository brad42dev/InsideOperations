Changes:
1. `state.rs`: Added `MfaPendingEntry` struct with `user_id`, `allowed_methods`, `expires_at` fields. Added `mfa_pending_tokens: Arc<DashMap<String, MfaPendingEntry>>` field to `AppState`, initialized in `AppState::new()`.

4. `handlers/ldap_auth.rs`: Inserts MFA gate after group role mapping and before JWT issuance — fetches `is_service_account` from DB, skips gate for service accounts.
5. `main.rs`: Registered new route `POST /auth/mfa/login-verify` → `handlers::mfa::mfa_verify_login`.

Note: The new endpoint is at `/auth/mfa/login-verify` (not `/auth/mfa/verify`) because `/auth/mfa/verify` is already taken by the TOTP enrollment verification handler (`verify_totp_enrollment` which requires JWT). The login-flow endpoint is public.
