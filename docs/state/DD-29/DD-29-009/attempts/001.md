## What Was Attempted

Added the `POST /api/auth/verify-password` endpoint to the auth-service. The endpoint:
- Reads user_id from `x-io-user-id` header (injected by gateway JWT middleware)
- Checks for SSO-only accounts (no password_hash) → 400 no_local_password
- Queries the most-recent active user_session for rate-limit counters
- Soft limit: 5 failures in 5-minute rolling window → 429 with retry_after_seconds
- Hard limit: 20 cumulative failures → session revoked, 401 forced_signout
- Wrong password: increments both soft+hard counters, returns 401 invalid_password + soft_remaining
- Correct password: resets all counters, clears locked_since, sets last_successful_unlock_at, returns 200 success:true
- Never issues new tokens

Also extended `GET /auth/me` to return `is_locked` (from active session locked_since IS NOT NULL) alongside the existing `auth_provider` field.

Created a migration adding 5 new columns to `user_sessions`.

Registered the route in both auth-service `main.rs` and api-gateway `main.rs`.
