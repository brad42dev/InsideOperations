## What Was Attempted

Fixed the `certs.map is not a function` crash in `frontend/src/pages/settings/Certificates.tsx`.

Root cause: The `/api/certificates` endpoint returns a `PagedResponse` (with `{ data: CertInfo[], pagination: {...} }`). The `api/client.ts` pagination branch detects `'pagination' in envelope && Array.isArray(envelope.data)` and wraps the result as `{ data: envelope.data, pagination: envelope.pagination }`. So `api.get<CertInfo[]>('/api/certificates')` was typed incorrectly ‚Äî the actual `data.data` at runtime was a `PaginatedResult` object (not `CertInfo[]`), causing `.map()` to throw.

Fix applied:
1. Changed the query's type parameter from `CertInfo[]` to `PaginatedResult<CertInfo>` so the type correctly reflects what client.ts returns.
2. Added `Array.isArray()` guard: `const certs: CertInfo[] = Array.isArray(rawData?.data) ? rawData.data : []` ‚Äî this safely defaults to `[]` if the response is non-array for any reason.
3. Added `PaginatedResult` to the import from `../../api/client`.
