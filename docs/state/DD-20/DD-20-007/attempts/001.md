## What Was Attempted

Updated the `io-offline` IndexedDB database in `useOfflineRounds.ts` to create all 5 spec-required stores. Bumped `DB_VERSION` from 1 to 2, renamed `STORE_NAME` from `pending-rounds` to `sync-queue`, and added migration logic in `onupgradeneeded` to delete the legacy `pending-rounds` store (if present from v1) and create all 5 stores: `sync-queue`, `rounds-data`, `media-blobs`, `point-cache`, `tile-cache`.

Created two new helper files:
- `frontend/src/shared/hooks/pointCacheDb.ts` — exports `cachePointValues(graphicId, values)` and `getPointCache(graphicId)` for reading/writing to the `point-cache` store. Each entry stores `{ id: graphicId, values, cachedAt }`.
- `frontend/src/shared/hooks/tileCacheDb.ts` — exports `cacheTile(graphicId, z, x, y, blob)`, `getTile(graphicId, z, x, y)`, and `evictLRU()`. LRU eviction caps at `MAX_GRAPHICS = 10` unique graphics and 70% of `navigator.storage.estimate()` quota. Records carry `accessedAt` (updated on reads too) and `sizeBytes` for accurate LRU ordering.

Updated `TileGraphicViewer.tsx` to use a custom Leaflet tile layer class (`OfflineTileLayerClass`) that extends `L.TileLayer` via `L.TileLayer.extend()`. The custom `createTile` method checks IndexedDB via `getTile()` before issuing a network request, and calls `cacheTile()` after a successful network fetch. `evictLRU()` is called once per layer mount (before the first write). Falls back gracefully to standard network loading on any error.
