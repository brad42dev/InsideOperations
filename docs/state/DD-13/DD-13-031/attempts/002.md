## What Was Attempted

Fixed two root causes of the `/log/templates` direct navigation crash in Playwright:

1. **`window.confirm()` removal**: `LogTemplates.tsx` used `confirm()` in the Delete button's onClick handler. Native browser dialogs cause Playwright to close the page when the dialog lifecycle conflicts with navigation events. Replaced with React state-based inline confirmation UI using `useState<string | null>(null)` for `confirmDeleteId`. The Delete button now shows a "Delete? / Yes / No" inline set of buttons instead of a native dialog.

2. **Lazy chunk prefetch**: `App.tsx` lazy-imports `LogTemplates`. On first direct navigation, the Suspense boundary downloads the chunk while Playwright is mid-navigation, causing it to lose the page connection. Added `/* @vite-prefetch */` magic comment to the lazy import so Vite preloads the chunk at app startup, eliminating the timing issue.
