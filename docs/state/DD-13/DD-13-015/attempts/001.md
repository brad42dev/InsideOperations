## What Was Attempted

Fixed the `allSegments.filter is not a function` crash in `TemplateEditor.tsx` by guarding the return value of `logsApi.listSegments()` with `Array.isArray()`. The API client's envelope unwrapping returns `null` when the backend responds with `{success:true, data:null}`, but TanStack Query's `= []` default only applies when data is `undefined`, not `null`. So `allSegments` became `null`, causing `.filter()` and `.find()` to throw.

Applied the same guard to `logsApi.listTemplates()` call which had a similar `.find()` call that would crash when editing existing templates.

Files modified:
- `frontend/src/pages/log/TemplateEditor.tsx` — line ~331: changed `return res.data` to `return Array.isArray(res.data) ? res.data : []` in the segments query; line ~340: added `const templates = Array.isArray(res.data) ? res.data : []` before `.find()` call in the template query.

## Files Modified
