## What Was Attempted

Added breadcrumb navigation to the ExpressionBuilder component so that when the insertion cursor is inside a nested container tile, a breadcrumb trail appears above the equation workspace showing the nesting path (e.g., "Root > (…) > round(1)"). Each item in the breadcrumb is clickable and dispatches SET_CURSOR to navigate back to that nesting level.

Two new constructs were added to `frontend/src/shared/components/expression/ExpressionBuilder.tsx`:

1. `buildBreadcrumbPath(tiles, targetParentId)` — a recursive helper that walks the tile tree from root to the container identified by `targetParentId`, collecting `{parentId, label}` items along the way. Returns an array prefixed with a "Root" item.

2. `BreadcrumbNav` component — renders a `<nav aria-label="Expression nesting path">` with one button per breadcrumb item. Returns null when `cursorParentId === null` (cursor at root). The last item is disabled (current level); all preceding items dispatch SET_CURSOR on click.

`BreadcrumbNav` is rendered just above the workspace `<div role="application">` inside the `ExpressionBuilder` render, between the DndContext palette and the workspace.
