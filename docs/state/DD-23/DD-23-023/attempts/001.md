## What Was Attempted

Fixed `WorkspaceTile.handleClick` in `frontend/src/shared/components/expression/ExpressionBuilder.tsx`. The handler previously dispatched `SET_CURSOR` with `parentId: null` unconditionally for every tile click, including container tiles. This caused `state.cursorParentId` to be reset to null whenever a user clicked a container tile, which made `BreadcrumbNav` (which returns null when `cursorParentId === null`) disappear.

Fix applied at line 742: for container tiles (`isContainer` or `isControlFlow`), dispatch `SET_CURSOR` with `parentId: tile.id` and `index: tile.children?.length ?? 0` (end of the container's children list). For non-container/leaf tiles, removed the `SET_CURSOR` dispatch entirely — leaf tile clicks only select the tile; cursor placement is handled by the `DropZoneRow` hit-box rows.

## Files Modified

- `frontend/src/shared/components/expression/ExpressionBuilder.tsx` — fixed `handleClick` in `WorkspaceTile` (lines 742–753): container tiles now move cursor into the container, leaf tiles no longer reset cursor to root.
