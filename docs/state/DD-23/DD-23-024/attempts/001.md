## Why This Attempt Succeeded

Root cause: the workspace `div` (role="application") had no `onKeyDown` handler and no `tabIndex`. When a tile with `tabIndex={0}` received focus and the user pressed ArrowLeft, the event bubbled past the workspace, past the dialog, and reached AppShell's window-level keyboard handler, which—combined with native ARIA focus movement on `role="option"` elements—caused focus to jump to a sidebar NavLink (leading to /rounds).

Fix: the `handleWorkspaceKeyDown` callback intercepts all arrow key events at the workspace level via event bubbling (tiles don't stopPropagation on arrow keys), calls preventDefault+stopPropagation, and dispatches SET_CURSOR to move the insertion cursor. The workspace div is also given `tabIndex={0}` so it can receive focus when the user clicks empty space.
