## What Was Attempted

Enhanced the visible insertion cursor in the expression builder workspace. DD-23-011 had already implemented a 2px blinking cursor, but UAT on 2026-03-24 still reported no visible cursor. Analysis showed three issues: (1) the 2px cursor was too thin and lacked visual contrast, (2) the empty-state only showed "Drop tiles here" text with the cursor visually adjacent but imperceptible, (3) hitboxes gave no hover feedback.

Changes made to `frontend/src/shared/components/expression/ExpressionBuilder.tsx`:

1. **Enhanced `renderCursor`**: Added `boxShadow: '0 0 4px var(--io-accent)'` glow, `borderRadius: '1px'`, `minWidth: '2px'` to prevent flexbox compression, changed from `aria-hidden="true"` to `role="presentation"` + `aria-label="Insertion cursor"` so screen readers can find it.

2. **Enhanced `renderHitbox`**: Added an inner `.io-hitbox-preview` div (position: absolute, 2px × 28px, opacity: 0) that becomes visible on hover via CSS `.io-hitbox:hover .io-hitbox-preview { opacity: 1 }`. This gives sighted users hover feedback showing where a click would place the cursor.

3. **Enhanced CSS injection**: Added the hover rule to the injected `<style>` block alongside the existing blink keyframe.

4. **Replaced empty-state "Drop tiles here" text**: The empty workspace now shows a flex row with: (a) a blinking 2px teal cursor with glow as a visual indicator, (b) instructional text "Click palette tiles to insert, or drag them here". The visible cursor in the empty state is the most prominent change — it makes the cursor position immediately perceivable when opening an empty expression editor.
