## What Was Attempted

Added a 5-level nesting depth limit for container tiles in ExpressionBuilder.tsx. The approach:

1. Added `showToast` import from `../Toast`.
2. Added `MAX_NESTING_DEPTH = 5` constant at module scope.
3. Added `CONTAINER_TILE_TYPES` array listing all container tile types: `['group', 'square', 'cube', 'round', 'negate', 'abs', 'if_then_else']`.
4. Added `getTileDepth(tiles, targetId, currentDepth)` recursive helper that traverses the tile tree and returns the depth of a tile by ID (-1 if not found).
5. In `handleAddFromPalette`: before creating/dispatching an INSERT_TILE for a container type, compute `parentDepth = getTileDepth(state.tiles, state.cursorParentId)` and block with a toast if `parentDepth + 1 >= MAX_NESTING_DEPTH`.
6. In `handleDragEnd`: after finding the tile locations, if the tile being dragged is a container type, compute `destDepth = getTileDepth(state.tiles, over.id)` and block with a toast if `destDepth >= MAX_NESTING_DEPTH`.
7. In `DropZoneRow`: added `depthBlocked = depth >= MAX_NESTING_DEPTH` flag, renders a red dashed border + subtle background + `data-depth-blocked="true"` attribute + tooltip when blocked.

Non-container tiles are unaffected at any depth.
