## What Was Attempted

Fixed `handleDragEnd` in `ExpressionBuilder.tsx` to correctly route palette-sourced drag drops into container tiles rather than always inserting at the top level.

Three specific changes:

1. Added `useDroppable` import from `@dnd-kit/core`.
2. Added `findParentId(tiles, childId)` helper function (after `findTileLocation`) that walks the tile tree and returns the owning container's ID (or null for top-level, undefined if not found).
3. Modified `DropZoneRow` to call `useDroppable` with id `container-zone-{parentId}` and attach the ref to the drop zone div. Empty container zones now show a blue highlight border when dragged over.
4. Rewrote the palette-source branch of `handleDragEnd` into 3 cases:
   - Case 1: `over.id` starts with `container-zone-` — the user dropped onto the DropZoneRow div directly (common for empty containers). Extract `parentId`, insert as last child.
   - Case 2: `over.id` is a container tile itself (in CONTAINER_TILE_TYPES) — insert as last child of that container.
   - Case 3: `over.id` is a non-container tile — use `findParentId` to find which parent array it belongs to, insert at the sibling position (preserving the pre-existing sibling-gap behavior).
