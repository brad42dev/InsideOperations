## What Was Attempted

Added clipboard (Ctrl+C / Ctrl+X / Ctrl+V) support to the ExpressionBuilder component.

Specifically:
1. Added `clipboard: ExpressionTile[] | null` field to `ExprBuilderState` interface and initialized it as `null` in the component's `useReducer` call.
2. Added three new action types to the `Action` union: `COPY_SELECTION`, `CUT_SELECTION`, `PASTE`.
3. Added `reassignIds(tiles: ExpressionTile[]): ExpressionTile[]` helper function after `cloneTiles`. It deep-clones the tile tree and replaces every `id` (including nested children, condition, thenBranch, elseBranch) with a new UUID via `newId()`.
4. Added three reducer cases in `exprReducer`:
   - `COPY_SELECTION`: walks the tile tree collecting tiles whose IDs are in `selectedIds`, deep-clones them with `cloneTiles`, stores result in `clipboard`. No tile mutation.
   - `CUT_SELECTION`: same collection + deep-clone into `clipboard`, then removes each selected tile from the cloned tree by `findTileLocation` + splice. Pushes prior tiles to `past`.
   - `PASTE`: calls `reassignIds(cloneTiles(state.clipboard))` to get fresh-ID copies, splices them into the child array at `cursorParentId`/`cursorIndex`, advances cursor. Pushes prior tiles to `past`.
5. Added `showCutConfirm` / `setShowCutConfirm` state variable.
6. Extended the keyboard `useEffect` handler to handle Ctrl+C (dispatch COPY_SELECTION), Ctrl+X (show cut confirmation dialog), Ctrl+V (dispatch PASTE).
7. Added a cut confirmation `Dialog.Root` in the JSX, using the same Radix UI Dialog pattern as the other confirmation dialogs in the file. The dialog shows the count of selected tiles and uses a danger-colored "Cut" button that dispatches `CUT_SELECTION` on confirm.
