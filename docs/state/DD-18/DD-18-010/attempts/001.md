Finding: The code was fully correct (route registered at `/settings` in archive-service, proxied via `/api/archive/*path` in api-gateway, frontend calling `/api/archive/settings`). 2. Killed old process (PID 43285), removed stale PID file
3. Started new archive-service (PID 3543806) from the rebuilt binary
4. Verified GET/PUT endpoints work through the gateway

No code changes were needed — the implementation was already correct in the source. The only gap was that the running binary had never been updated after the code was written.
