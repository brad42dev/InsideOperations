## What Was Attempted

The archive service GET/PUT /settings endpoints and gateway proxy routing were already implemented in the DD-18-007 commit (c21c348). The root cause of the UAT failure was twofold:

1. The UAT was run before the DD-18-007 backend implementation was deployed (the route didn't exist at test time).
2. A secondary bug was introduced: the settings.rs handler used wrong SQL table name "system_settings" instead of the actual DB table name "settings" (created in migration 20260314000008_settings_audit.up.sql). This would cause PUT /api/archive/settings to return 500 at runtime.

Fixed the table name bug in services/archive-service/src/handlers/settings.rs by replacing all three "system_settings" references with "settings". Also updated comment in main.rs.
