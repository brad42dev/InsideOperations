Fixed three bugs in the batch shapes endpoint:

1. **Route URL fix** in `services/api-gateway/src/main.rs`: Changed route registration from `/api/shapes/batch` to `/api/v1/shapes/batch` (and `/api/stencils/batch` to `/api/v1/stencils/batch` for consistency).

Response now returns a map keyed by shape_id with `{ svg, sidecar }` per shape, extracting the sidecar from `metadata->'sidecar'`.

3. **Sidecar data fix** in `services/api-gateway/src/seed_shapes.rs`: Added `sidecar: &'static str` field to `ShapeSeed` struct. Updated `seed_shape_library` to parse the sidecar JSON and embed it in the `metadata` JSONB column under a `"sidecar"` key. Added full sidecar JSON strings sourced from `frontend/dist/shapes/{category}/{shape-id}.json` to all 65 ShapeSeed entries (including agitator and support composable parts with their minimal part sidecars). For shapes without a dist JSON file (`column-distillation-narrow-trayed`, `column-distillation-wide-trayed`), constructed sidecars using the same pattern as their plain variants.
