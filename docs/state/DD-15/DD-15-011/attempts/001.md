## What Was Attempted

Fixed the TypeError crash in EditRoleDialog in `frontend/src/pages/settings/Roles.tsx`.

Root cause: When the Edit button is clicked, `handleEdit(role)` sets `editRole = role as RoleDetail` and `editOpen = true`. The `roleDetailQuery` is triggered but begins as loading (`data = undefined`). The `EditRoleDialog` receives `role={roleDetailQuery.data ?? editRole}`, which evaluates to `editRole` (a `Role` object cast to `RoleDetail`). The `useEffect` then ran `role.permissions.map((p) => p.name)` — but a `Role` has no `permissions` field, so `role.permissions` was `undefined`, and `.map()` on `undefined` threw `TypeError: Cannot read properties of undefined (reading 'map')`.

Changes made:
1. **Guarded the `.map()` call**: Changed `role.permissions.map(...)` to `(role.permissions ?? []).map(...)` in the `useEffect`. This prevents the crash when `permissions` is `undefined` during the initial render before `roleDetailQuery` resolves.
2. **Added `isLoading` prop to `EditRoleDialog`**: When the detail query is loading, a "Loading role details…" message is shown and the form is hidden (`display: none`), preventing interaction with incomplete data.
3. **Passed `isLoading={roleDetailQuery.isLoading}` from `RolesPage`** to the `EditRoleDialog`.
