## What Was Attempted

Created three new per-platform DCS parser modules in `services/parser-service/src/handlers/`:

1. `dcs_ge.rs` ‚Äî GE iFIX SVG parser. Reads SVG, extracts `data-tag`/`data-symbol`/`data-type`/`data-class` attributes as tag bindings. Optionally parses a `.xtg` XML sidecar (tag-group definitions). Provides `ZipContents` struct shared by all three parsers.

2. `dcs_rockwell.rs` ‚Äî Rockwell FactoryTalk XML parser. Reads XML with `<gfx>` root element using `roxmltree`. Maps FactoryTalk object types (AnalogDisplay, BarGraph, ValveSymbol, etc.) to intermediate format `element_type`, `source_type`, and `display_element_hint`. Extracts tag bindings from `tagname` attribute.

3. `dcs_siemens.rs` ‚Äî Siemens WinCC Unified SVG parser. Parses SVG geometry, scans every element for `siemens:*` namespace attributes, extracts tag from `siemens:tagName`, strips namespace prefix when storing in properties bag.

Updated `handlers/mod.rs` to declare all three new modules.

Updated `handlers/dcs_import.rs` to:
- Import the three modules and the shared `ZipContents` struct
- Collect `.xtg` and `.xml` files during the ZIP scan pass
- Build a `ZipContents` struct after the scan
- Dispatch `parse_zip()` by platform: `ge_proficy` ‚Üí `dcs_ge::parse`, `rockwell_factorytalk` ‚Üí `dcs_rockwell::parse`, `siemens_wincc_unified` ‚Üí `dcs_siemens::parse`, `generic_svg` ‚Üí existing SVG path, `generic_json` ‚Üí existing JSON path, `_` ‚Üí `stub_result`
- Added `siemens_wincc_unified` to `is_valid_platform()` and `platform_display_name()` (it was missing)
- Added `svg_display_dimensions()` helper to extract width/height from an SVG in the ZIP
