## What Was Attempted

Implemented the full 6-tab System Health page and shell status dot popover in two phases:

**Phase 1 — SystemHealthDot.tsx:**
- Replaced the non-interactive colored dot with a button-triggered click-to-open popover (custom div, no new packages needed)
- Popover displays three rows: WebSocket status (from `wsManager.onStateChange`), OPC Data status (from `/api/opc/sources/stats`), Server reachability (from `/api/health/services` aggregate)
- Added "Last update: X seconds ago" timestamp
- Added "Open System Health" button visible only when user has `system:monitor` permission via `usePermission()`
- Added CSS keyframe pulse animation (`io-dot-pulse`) injected once into `document.head`, triggered via class toggle on `dotColor` state transition
- Color logic: Green (all healthy + WS connected), Yellow (degraded services or WS connecting), Red (WS disconnected >30s OR all OPC offline), Gray (unknown)
- Popover closes on outside click via `document.addEventListener('mousedown')`

**Phase 2 — SystemHealth.tsx:**
- Replaced single-tab Services table with 6-tab layout using custom CSS-based tabs (no new Radix packages needed)
- Services tab: expanded to 8 spec columns (Service, Status, Uptime, Version, p50/p95 ms, Req/s, Error%, Last Check), hits `/api/health/services/detail` with fallback to base health endpoint, auto-refreshes every 15s
- Database tab: summary tiles (migration version, db size, TimescaleDB version, compression ratio, replication lag) + connection pool table from `/api/health/database`
- OPC Sources tab: per-source cards with connection status, subscribed points, update rate, last update, reconnection count, and recent error history from `/api/opc/sources/stats`
- WebSocket tab: metric tiles (active connections, subscriptions, message rate, backpressure events, avg duration) from `/api/health/websocket`
- Jobs tab: 4 category sections (Email/Alerts/Exports/Imports) each showing their counter columns from `/api/health/jobs`
- Metrics tab: 6 uPlot TimeSeriesChart charts (Request Rate, Latency p95, Error Rate, WS Connections, OPC Update Rate, DB Pool Utilization) from `/api/health/metrics?metric=...&from=...&to=...` with Last 1h/6h/24h/7d/30d time range selector; no hardcoded data
