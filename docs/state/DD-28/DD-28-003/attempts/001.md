## What Was Attempted

Implemented all four missing email provider adapters in `services/email-service/src/queue_worker.rs`:

1. **MS Graph** (`"msgraph"`): `send_msgraph` function using `reqwest` to POST to `https://login.microsoftonline.com/{tenant}/oauth2/v2.0/token` for OAuth2 client-credentials token, then POST to `https://graph.microsoft.com/v1.0/users/{send_as_user}/sendMail`. Token cached with 5-minute pre-expiry refresh.

2. **Gmail** (`"gmail"`): `send_gmail` function using `jsonwebtoken` to create RS256 JWT for service-account impersonation, exchanges it for a Google access token via `https://oauth2.googleapis.com/token`, builds RFC 2822 message with `lettre::Message`, base64url-encodes it, then POSTs to `https://gmail.googleapis.com/gmail/v1/users/{send_as_user}/messages/send`.

3. **SES** (`"ses"`): `send_ses` function deriving SMTP credentials from AWS `access_key_id`/`secret_access_key` via HMAC-SHA256 key derivation (matching AWS SES spec), then routing through `lettre` SMTP to `email-smtp.{region}.amazonaws.com:587`.

4. **SMTP+XOAUTH2** (`"smtp_xoauth2"` or `"smtp"` with `auth_method="xoauth2"`): within `send_smtp`, when `auth_method == "xoauth2"`, acquires token via client-credentials flow using `acquire_client_credentials_token`, then passes to lettre with `Mechanism::Xoauth2`.
