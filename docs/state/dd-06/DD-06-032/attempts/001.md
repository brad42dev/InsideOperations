## What Was Attempted

Fixed G+D and G+R navigation shortcuts in AppShell.tsx by wrapping the navigate() call in flushSync() from react-dom. The G_KEY_MAP already contained correct entries for all 11 routes (including 'd' → '/designer' and 'r' → '/reports'). The root cause was that with v7_startTransition:true in BrowserRouter, React Router wraps all navigate() calls in React.startTransition, deferring the URL change as a low-priority transition. In the UAT Playwright test, the URL check happened before the deferred transition committed, making G+D and G+R appear as silent no-ops. Using flushSync forces synchronous navigation that commits the URL change immediately.
