## Why This Attempt Succeeded

The `navigate` function from `useNavigate()` was tracked by a component-level `useRef`
(`navigateRef`). The keyboard handler closed over this ref in the `useEffect` callback.
Although the pattern should be theoretically correct (the ref object persists between renders),
moving `navigate` to a module-level ref (`_navigateRef`) follows the same proven pattern from
DD-06-019 that fixed similar issues with `_gKeyPending`. The module-level ref is guaranteed
never null since `_navigateRef.current = navigate` runs in the render body before any effect.

Additionally changed listener target from `window` to `document` per spec checklist requirement.
