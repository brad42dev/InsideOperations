## What Was Attempted

Added XLSX upload support and conflict detection to the Bulk Update workflow.

**XLSX support:**
- Added `calamine = "0.26"` to `services/api-gateway/Cargo.toml`
- Replaced `extract_csv_and_target()` with `extract_file_and_target()` in `bulk_update.rs`, which detects XLSX via content-type or filename extension and uses `calamine::open_workbook_from_rs::<Xlsx<_>, _>()` to parse XLSX into `Vec<HashMap<String, String>>`
- Changed `accept` attribute on file input in `BulkUpdate.tsx` to include `.xlsx` and the XLSX MIME type
- Added a `parse_xlsx_bytes()` helper that converts calamine `Data` enum variants to strings

**Conflict detection:**
- Added `_exported_at` as the first column in all `csv_headers()` strings for all 10 target types
- Added `exported_at_str = Utc::now().to_rfc3339()` in `get_template` handler and prefixed it to each data row format string
- Added `fetch_updated_at_map()` helper function that fetches `id → updated_at` per target type (returns empty map for types without `updated_at` column)
- Extended `compute_diff()` to accept `db_updated_at: &HashMap<String, DateTime<Utc>>` and `exported_at: Option<DateTime<Utc>>` — rows where `db_updated_at > exported_at` go into a new `conflicted: Vec<ModifiedRow>` list instead of `modified`
- Added `conflicted: Vec<ModifiedRow>` to `DiffPreview` struct
- The `apply_bulk_update` handler reads `conflict_resolution` from the multipart extra fields — "overwrite" applies conflicted rows; anything else (default "skip") adds them to `failed_rows` with `reason_type: "skipped_conflict"`

**Frontend:**
- Added `conflicted: ModifiedRow[]` to `DiffPreview` TypeScript interface
- Added `ConflictedSection` component that renders conflicted rows with ⚠ warning badge, explains the conflict, and provides a radio group: "Skip conflicted rows" (default) vs "Overwrite with template values"
- Added `conflictResolution` state to `BulkUpdateTab`
- Updated `hasChanges` to include `conflicted.length > 0`
- Updated `apply` API function to accept and forward `conflictResolution` parameter
