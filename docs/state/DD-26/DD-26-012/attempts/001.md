## What Was Attempted

Changed `DomainSlot.session` from `Option<Arc<RwLock<Arc<ort::session::Session>>>>` to
`Arc<RwLock<Option<Arc<ort::session::Session>>>>` — the double-Arc hot-swap pattern where the
outer `Arc<RwLock<...>>` is always present (enabling swap at any time), and the inner `Option`
holds the session pointer (None in Phase 1, Some in Phase 2 when ONNX is wired up).

Added `ModelManager::swap_domain(domain, new_session)` — acquires a write lock on the target
domain's session RwLock, replaces the inner `Option` with `Some(new_session)`, and releases
the lock. Old session Arcs are dropped when the last in-flight inference clone is released.

Added `ModelManager::session_for_domain(domain)` — acquires a read lock, clones the inner
`Option<Arc<Session>>`, and releases the lock. Used by `run_inference()` to hold only the
cloned Arc during inference, allowing concurrent write-lock swaps to proceed.

Updated `run_inference()` in `main.rs` to call `session_for_domain()` — acquiring the read
lock, cloning the inner Arc, dropping the lock before inference runs.

Updated `get_status()` to include `model_version`, `model_id`, and `session_active` fields
per domain, so uploading a new model is reflected in the status response for that domain only.

Made `DomainSlot::is_active()` async (now reads the session lock to check for Some).
Updated `ModelManager::domain_is_loaded()` to async accordingly.
