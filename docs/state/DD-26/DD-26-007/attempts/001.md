## What Was Attempted

Created `frontend/src/pages/designer/components/RecognitionWizard.tsx` as a three-step modal wizard:

- Step 1 (UploadStep): file picker accepting `.png,.jpg,.jpeg,.pdf,.tif,.tiff` with drag-and-drop support. Calls `recognitionApi.runInference()` (which posts to `POST /api/recognition/detect`) with `{ domain: "auto" }`. Shows spinner while loading.
- Step 2 (ReviewStep): renders uploaded image with an SVG overlay of bounding boxes. Each detection renders a `<rect>` with stroke color coded by confidence (green #22c55e for >0.90, yellow #eab308 for 0.70-0.90, red #ef4444 for <0.70). Per-detection accept/reject/correct controls. Correct opens a dropdown populated from `GET /api/recognition/classes?domain={domain}`. Domain override selector (P&ID / DCS) with live reload of class list on change.
- Step 3 (GenerateStep): summary of accepted/corrected/rejected counts, "Generate Graphic" button calls `recognitionApi.generateGraphic()` (POST /api/recognition/generate). On success navigates to `/designer/graphics/{graphic_id}/edit`. Submits corrections feedback via `recognitionApi.submitCorrections()` (fire-and-forget).

Also exported `RecognitionWizardTrigger` component that:
- Checks `GET /api/recognition/status` on mount
- If both domains show `mode: "disabled"`, renders a "Recognition not available" notice instead of the button
- If user lacks `designer:import` permission (checked by caller), renders nothing
- Otherwise renders a styled quick-action button that opens the wizard modal

Integrated `RecognitionWizardTrigger` into `frontend/src/pages/designer/DesignerHome.tsx`:
- Added `useDesignerPermissions` import and `RecognitionWizardTrigger` import
- Added `const perms = useDesignerPermissions()` in the component
- Added `<RecognitionWizardTrigger canImport={perms.canImport} renderAs="button" />` in the Quick Actions section alongside the existing Symbol Library and Import DCS Graphics buttons
