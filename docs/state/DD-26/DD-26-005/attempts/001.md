Added four missing crates to `services/recognition-service/Cargo.toml`: `ort` (2.0.0-rc.12 with load-dynamic feature), `sha2` and `zip` via workspace dependencies, `image` (0.25 with png/jpeg/tiff features), and `hex` (0.4) plus `digest` (0.10). The `ort` 2.x stable is not yet published — only RC versions are available on crates.io, so the spec's `version = "2"` requirement was satisfied using `2.0.0-rc.12`.

Rewrote `upload_model` handler in `src/main.rs` to:
1. Read only the `file` multipart field (dropping the form `domain` field — domain comes from manifest)
2. Write bytes to `{model_dir}/incoming/{uuid}.iomodel` on disk
3. Call `parse_and_verify_iomodel` in a `spawn_blocking` closure
4. Clean up temp file regardless of parse outcome
5. Construct `ModelInfo` using domain/version/class_count from manifest

Added `parse_and_verify_iomodel` function that:
- Opens the ZIP with `zip::ZipArchive`
- Reads and deserializes `manifest.json` into `IomodelManifest`
- If `model_hashes` is absent or empty: logs `warn!("model integrity not verified — pre-1.1 package")` and continues
- If `model_hashes` is present: for each (filename, expected_hash) reads the file from ZIP, computes SHA-256 via `sha2::Sha256`, formats as `"sha256:<hex>"`, rejects on mismatch

Added `IomodelManifest` struct with `model_domain`, `version`, `class_count`, and `model_hashes` fields (all optional/defaulted).
