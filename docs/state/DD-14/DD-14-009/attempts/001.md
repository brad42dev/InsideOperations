## What Was Attempted

Fixed the crash in `frontend/src/pages/rounds/index.tsx` caused by calling `.map()` on non-array values. The API client's `request()` function in `client.ts` unwraps envelope `data` fields without checking if the result is an array — when the API returns `{ "data": null }` or `{ "data": {} }`, the unwrapped value is passed through as-is. Then `RoundsPage` extracts data with `availableResult?.success ? availableResult.data : []` which gets `null` or an object (not an array), causing `.map()` to crash.

Two changes were made to `frontend/src/pages/rounds/index.tsx`:

1. Added `type RoundTemplate` and `type RoundSchedule` to the import from `../../api/rounds` (needed for explicit type annotations on the extracted arrays).
2. Changed all five data extraction lines (pendingInstances, inProgressInstances, historyEntries, templates, schedules) to add `Array.isArray()` guards before using `.data`, so that a non-array API response is safely treated as an empty array instead of causing a crash.
