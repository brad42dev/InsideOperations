## Why This Attempt Succeeded

Root cause: The API client's envelope unwrapper (client.ts lines 135-145) returns `envelope.data as T`. If the backend returns a paginated shape (`{ data: [...], pagination: {...} }`), the client wraps it as `{ data: [...], pagination: {...} }` and `success: true`. But for non-paginated endpoints like `/api/rounds/templates`, if the backend wraps in an outer `{ success: true, data: [...] }` envelope, the client returns `{ success: true, data: [...] }` correctly. However, if the backend response has any unexpected shape, `data` could be an object rather than an array.

The fix adds `Array.isArray()` guards consistent with how `index.tsx` already handles the same query results. This prevents `.map()` from crashing when `data` is anything other than an array.
