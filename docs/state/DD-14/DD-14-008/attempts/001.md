## What Was Attempted

Created a `PrintDialog` component at `frontend/src/pages/rounds/PrintDialog.tsx` implementing the full printable round checklist feature with two modes (blank and current_results). The dialog uses Radix UI Dialog primitive with a template selector, mode radio group (Blank checklist / Current results), and page size selector (Letter / A4). Added a Print button to the Rounds page header toolbar (always visible across all tabs) and per-template Print buttons in RoundTemplates.tsx.

The PrintDialog renders a hidden `<div id="rounds-print-area">` that becomes visible only via `@media print`. It contains:
- Header with template name, print date, mode label, and blank Operator/Shift/Date lines
- An 8-column table: # | Equipment ID | Location | Description | Expected Range | Reading | Pass/Fail | Notes
- For numeric checkpoints: Expected Range shows HH/H/L/LL values (alarm mode) or min-max (limit mode) with unit
- For dropdown checkpoints: Expected Range shows options joined with " / "
- For multi_field checkpoints: expanded to one row per sub-field, grouped with thicker borders
- In current_results mode: fetches latest completed round instance and pre-fills Reading and Pass/Fail columns
- Footer with "UNCONTROLLED COPY" watermark and page number
- @media print CSS injected into <head> hiding all page content except #rounds-print-area
