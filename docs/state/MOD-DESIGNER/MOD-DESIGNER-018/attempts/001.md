## What Was Attempted

### Part 1 ‚Äî Canvas boundary visual (DesignerCanvas.tsx)

Inside the scene-graph `<g transform="translate(panX,panY) scale(zoom)">`, inserted a boundary visual as the first child element. The visual:

- Computes `boundaryVisible` from `panX`, `panY`, `zoom`, `canvasW`, `canvasH`, and container rect width/height (from `containerRef.current?.getBoundingClientRect()`).
- When `boundaryVisible` is false: renders nothing.
- When `autoHeight=true`: renders a `<path>` for top/left/right edges only (SVG path `M0,canvasH L0,0 L canvasW,0 L canvasW,canvasH`) plus a `<line>` horizontal guide at `y=canvasH`.
- Normal mode: renders a `<rect x=0 y=0 width=canvasW height=canvasH>` with dashed stroke.
- All stroke widths and dash arrays divided by `zoom` so they remain 1px visually at any zoom level.
- Guarded by `!testMode` to suppress in test/view-only mode.

Also exported `getNodeBounds` as a named export from DesignerCanvas.tsx.

### Part 2 ‚Äî Out-of-bounds save warning toast (index.tsx)

- Added `{ explicit?: boolean }` parameter to `handleSave` (default `false`).
- After a successful save, if `explicit=true`: iterates `currentDoc.children`, calls `getNodeBounds(node)` for each, collects IDs where the node extends outside `[0,0,cw,ch]`.
- If any out-of-bounds: calls `showToast({ title: "N element(s) outside the canvas boundary", variant: 'warning', duration: 8000, action: { label: 'Select', onClick: () => setSelectedNodes(outOfBoundsIds) } })`.
- Created `handleExplicitSave = () => handleSave({ explicit: true })` for toolbar and mode tabs.
- Updated `DesignerToolbar onSave` and `DesignerModeTabs onSave` props to use `handleExplicitSave`.
- Updated Ctrl+S handler to call `handleSave({ explicit: true })`.
- Auto-save (IndexedDB interval) writes directly to IDB without calling `handleSave` ‚Äî no toast triggered.
- Imported `getNodeBounds` from `./DesignerCanvas` in index.tsx.
