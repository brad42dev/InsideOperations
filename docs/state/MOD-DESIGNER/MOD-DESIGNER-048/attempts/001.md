## Why This Attempt Succeeded

The root cause was a missing `focus()` call when drag starts. The Escape cancel logic was already fully implemented and correct. Clicking an SVG child element inside the container div does not automatically focus the container div in browsers, even when `tabIndex={0}` is set. Without focus, the React `onKeyDown` event handler on the container div never fires. Adding `containerRef.current?.focus({ preventScroll: true })` when `inter.type = 'drag'` is set resolves the issue with a minimal, targeted change.
