## What Was Attempted

Fixed canvas drag-to-move in DesignerCanvas.tsx by addressing two root causes:

1. Added document-level `mousemove`/`mouseup` listeners when a canvas element drag starts
   (same pattern as `startGuideCreate`). The `mouseup` uses `{ capture: true }` so it fires
   before React's synthetic event handler, guaranteeing MoveNodesCommand is committed regardless
   of where the pointer lands at drag end (over child elements, outside container, etc.).

2. Added `dragActive` prop to `SelectionOverlay` component and two state tracking variables
   (`canvasDragActiveRef` for closures, `isDraggingCanvas` for React re-renders). When
   `dragActive=true`, all resize handle `<rect>` elements and the rotate handle `<circle>` get
   `pointer-events: none`, eliminating the "subtree intercepts pointer events" Playwright error.

## Files Modified

- `frontend/src/pages/designer/DesignerCanvas.tsx`:
  - Added `canvasDragActiveRef = useRef(false)` and `isDraggingCanvas` state
  - Added `dragActive?: boolean` prop to `SelectionOverlay` component
  - Updated resize handle `<rect>` and rotate handle `<circle>` to use `pointer-events: dragActive ? 'none' : 'all'`
  - After `inter.type = 'drag'` in `handleMouseDown`: set `canvasDragActiveRef.current = true`, `setIsDraggingCanvas(true)`, attach `dragMoveFn` to `document.mousemove` and `dragUpFn` to `document.mouseup` (capture)

  - Pass `dragActive={isDraggingCanvas}` to `SelectionOverlay` at call site

## Why This Attempt Succeeded

Both root causes were addressed:
1. Document-level capture mouseup listener guarantees MoveNodesCommand commit even when the native
   mouseup event fires over a child SVG element or outside the React container.
2. Suppressing pointer-events on selection overlay handles during drag eliminates the Playwright
   "subtree intercepts pointer events" error and prevents any accidental resize-start during a move.

The double-commit race is avoided: dragUpFn uses { capture: true }, so it fires and sets
inter.type = 'none' before React's onMouseUp handler runs — the existing drag branch
`if (inter.type === 'drag' && d)` naturally skips since the type was already cleared.
