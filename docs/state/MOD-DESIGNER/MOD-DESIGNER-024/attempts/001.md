## What Was Attempted

Extended the Designer tab system to support group sub-tabs. The approach:

1. **tabStore.ts** — Extended `DesignerTab` interface to add `type: 'graphic' | 'group'`, `groupNodeId?: string`, `groupName?: string`. Added three new store actions: `openGroupTab()`, `closeGroupTabsForGraphic()`, `findGroupTab()`. Updated `findTabByGraphicId()` to only match `type === 'graphic'` tabs.

2. **DesignerCanvas.tsx** — Added `onOpenGroupInTab` and `groupSubTabNodeId` props to `DesignerCanvasProps`. Wired the "Open in Tab" context menu item (was disabled/stub) to call `onOpenGroupInTab`. Added `GroupTabIndicatorOverlay` component that renders a small accent-colored badge on group nodes that have open sub-tabs. Added scene rendering filter: when `groupSubTabNodeId` is set, only renders that group's children. Added `useTabStore` subscription to compute `openGroupTabNodeIds` set.

3. **DesignerTabBar.tsx** — Added `tabLabel()` function that formats group sub-tab labels as `[Graphic] › [Group]` (each truncated to 12 chars). Updated `TabItem` to use `displayLabel` and `fullTitle` for label, title, and aria attributes.

4. **index.tsx** — Added `tabStoreOpenGroupTab`, `tabStoreCloseGroupTabs`, `setActiveGroup` subscriptions. Added `activeTab` derived state. Added `openGroupInTab()` callback that: saves outgoing viewport, computes fit-to-group viewport using window dimensions, calls `tabStoreOpenGroupTab`, applies viewport and sets `activeGroupId`. Updated `switchToTab()` to handle group sub-tabs (no server fetch, just viewport + activeGroup restore) and clear `activeGroupId` when switching to graphic tabs. Updated `requestCloseTab()` to skip save prompt for group tabs, and to close all group sub-tabs before closing a parent graphic tab. Updated `isDirty` sync to mark parent graphic tab modified when editing in a group sub-tab. Passed `onOpenGroupInTab` and `groupSubTabNodeId` props to `DesignerCanvas`.
