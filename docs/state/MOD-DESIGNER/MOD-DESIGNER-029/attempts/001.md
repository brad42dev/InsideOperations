## What Was Attempted

Root-cause analysis revealed that MOD-DESIGNER-023 introduced a `useParams()` regression in
`frontend/src/pages/designer/index.tsx`. The original code correctly used
`const { id: graphicId } = useParams<{ id: string }>()` to extract the `:id` route parameter
(since App.tsx routes use `designer/graphics/:id/edit`). MOD-DESIGNER-023 changed this to
`const { graphicId } = useParams<{ graphicId?: string }>()` which always yields `undefined`
because there is no `:graphicId` param in any route â€” only `:id`.

With `graphicId` always undefined:
- `isNew` is always `true`
- `loadDoc` calls `setShowNewDialog(true)` immediately instead of loading the graphic
- The route sync effect (`if (isNew || !graphicId) return`) always returns early â€” no tab is
  ever opened via `openGraphicInTab`
- `DesignerTabBar` renders `null` because `tabs.length === 0`

The fix: restore the correct `useParams` extraction:
```ts
const { id: graphicId } = useParams<{ id?: string }>()
```

Also corrected the stale route comment in the JSDoc header from `:graphicId` to `:id`.
