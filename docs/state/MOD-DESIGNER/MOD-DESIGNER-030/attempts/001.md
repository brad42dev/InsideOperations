## What Was Attempted

Root cause identified: the Designer page `loadDoc()` function (called on mount when navigating directly to `/designer/graphics/:id/edit`) loads the graphic into `sceneStore` but never calls `tabStoreOpenTab()`. Since `DesignerTabBar` returns `null` when `tabs.length === 0`, no tab bar appeared.

Two code paths were fixed:
1. `loadDoc()` in `frontend/src/pages/designer/index.tsx` — added `tabStoreOpenTab(record.id, record.name)` after successful graphic fetch
2. `handleNewConfirm()` in the same file — added `tabStoreOpenTab('new', name)` after creating a new document in memory
3. `handleSave()` — added upgrade of the 'new' placeholder tab to the real server graphicId once a new graphic is first saved
4. `tabStore.ts` — added `setTabGraphicId(tabId, graphicId, graphicName?)` action to support upgrading the placeholder tab

## Files Modified

- `frontend/src/pages/designer/index.tsx` — added `tabStoreOpenTab` calls in `loadDoc()` and `handleNewConfirm()`; added `tabStoreSetGraphicId` hook and call in `handleSave()`; updated dependency arrays
- `frontend/src/store/designer/tabStore.ts` — added `setTabGraphicId` action to interface and implementation
