## What Was Attempted

Added group proportional resize (Figma model) to `DesignerCanvas.tsx` in two parts:

1. **`getNodeBounds()` group case** — Added an `if (node.type === 'group')` branch that computes the union bounding box of all children by calling `getNodeBounds()` recursively on each child. Empty groups fall back to a 64x64 placeholder. This is required for resize handles to render at the correct bounding box positions.

2. **Resize mouseup commit block** — Added `else if (target?.type === 'group')` branch after the `stencil` case. The algorithm:
   - Enforces minimum 20x20 group size (`Math.max(20, nw/nh)`)
   - Computes scale factors `sx = finalW / origBBox.w`, `sy = finalH / origBBox.h` (guarded against divide-by-zero)
   - Skips pipe children (as specified — pipes use endpoint dragging)
   - For each remaining child: computes new canvas position using relative offset scaled by sx/sy, and new scale by multiplying existing scale by sx/sy
   - For nodes with explicit dimensions: `image` uses `ResizeNodeWithDimsCommand` with `['displayWidth', 'displayHeight']` keys; `widget`, `text_block`, `stencil`, `embedded_svg` use `ResizeNodeWithDimsCommand` with default `['width', 'height']` keys
   - For primitives, symbol_instances, nested groups: uses `ResizeNodeCommand` (transform-only)
   - Wraps everything (group position update + all child commands) in a single `CompoundCommand('Resize Group', ...)`

No recursion into nested groups (as specified).
