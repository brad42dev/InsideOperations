## What Was Attempted

Extended `DesignerCanvas.tsx` to support multi-node resize via proportional scaling of all selected nodes within their union bounding box.

**Changes made:**

1. **`interactionRef` type** — Added four new fields: `resizeNodeIds: NodeId[]`, `resizeNodeOrigTransforms: Map<NodeId, Transform>`, `resizeNodeOrigDims: Map<NodeId, { w: number; h: number }>`, `resizeOrigSelectionBBox: { x, y, w, h }`.

2. **`startResize` function** — Extended signature with optional `allNodeIds?: NodeId[]` and `selectionBBox?` params. When multi-node, iterates all IDs, filters out pipes, captures each node's original transform and `getNodeBounds` dimensions into the new Maps. Falls back to single-entry Maps for single-node case.

3. **`SelectionOverlay` component** — Changed `showResizeHandles = isSingle && !!onResizeStart` to `showResizeHandles = !!onResizeStart`. Added union bounding box computation at component level. For single selection: draws handles on per-node bounds (unchanged logic). For multi-selection: draws handles on the union bbox once; `onMouseDown` passes `allNodeIds` and `selectionBBox` to `onResizeStart`.

4. **Mouseup resize commit block** — Introduced `buildResizeCommand(node, origT, newPos, rawW, rawH, origBoundsRef?)` helper that encapsulates all per-node-type command creation (primitive, image, widget, embedded_svg, annotation, symbol_instance, text_block, stencil, display_element, group). Added multi-node path: when `inter.resizeNodeIds.length > 1`, computes selection-level scale factors, iterates all participating nodes, calls `buildResizeCommand`, wraps results in `CompoundCommand('Resize', perNodeCommands)`. Single-node path uses the same `buildResizeCommand` helper.
