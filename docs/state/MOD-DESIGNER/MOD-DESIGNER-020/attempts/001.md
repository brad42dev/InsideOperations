## What Was Attempted

Implemented auto-height scrollable canvas for Report and Dashboard modes in DesignerCanvas.tsx and CanvasPropertiesDialog.tsx.

Specifically:
1. Replaced the single `canvasH = doc?.canvas.height ?? 1080` line in DesignerCanvas.tsx with:
   - `declaredH` for the raw canvas.height value
   - `autoHeightEnabled` flag from `doc?.canvas.autoHeight ?? false`
   - `contentBoundingBoxBottom` useMemo that iterates `doc.children`, calls `getNodeBounds()` on each visible node, and returns the maximum `y + h`
   - `canvasH` computed as `Math.max(declaredH, contentBoundingBoxBottom + 80)` when autoHeightEnabled, else `declaredH`

2. Fixed the canvas boundary visual code to use `declaredH` (not `canvasH`) for the page guide line and edge path endpoints when `autoHeightEnabled`. Previously this code used `canvasH` which would have moved the guide line to the auto-computed height rather than the declared minimum.

3. Updated CanvasPropertiesDialog.tsx:
   - Height input label changed from `isReport && autoHeight ? 'Min. Height' : 'Height'` to `autoHeight ? 'Min. Height / Page Height' : 'Height'` (applies to any mode, not just report)
   - Height input tooltip updated to full spec text: "The canvas grows automatically to fit content. This value sets the minimum height."
   - Removed the `disabled` state on the height input (the spec doesn't say to disable it, just relabel it)
   - Auto-grow checkbox now visible for both report and dashboard modes (`isReport || designMode === 'dashboard'`)

The `autoHeight?: boolean` field in graphics.ts was already present from a prior task.
