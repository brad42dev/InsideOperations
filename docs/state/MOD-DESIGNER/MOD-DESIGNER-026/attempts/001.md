## What Was Attempted

Fixed the TypeError crash in `DesignerGraphicsList.tsx` caused by unsafe access of `data.data.data` when the API client returns the data array directly (no `pagination` key in server response).

Root cause: `client.ts` has two envelope-unwrapping paths:
1. When the server response includes a `pagination` key AND array `data`: wraps as `{ data: <array>, pagination: {...} }` — so `data.data` is `{ data: [...], pagination }` and `data.data.data` is the array.
2. When the server response has `data` but no `pagination` key: unwraps directly and returns `envelope.data` — so `data.data` IS the array itself and `data.data.data` is `undefined`.

When `data.data.data` is `undefined`, `allGraphics` becomes `undefined`, then `filtered` (from `useMemo`) becomes `undefined`, then `filtered.slice()` throws "Cannot read properties of undefined (reading 'slice')".

Fix applied: Made the array extraction defensive to handle both API shapes. Now checks `Array.isArray(data.data)` first — if it IS an array, use it directly; otherwise fall back to accessing `.data` on it (paginated shape) with a `?? []` guard.
