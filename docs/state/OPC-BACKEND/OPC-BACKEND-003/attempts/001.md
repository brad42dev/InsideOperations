## What Was Attempted

Modified `services/opc-service/src/driver.rs` to apply PercentDeadband 1% DataChangeFilter for
AnalogItemType nodes that have a valid EURange, and AbsoluteDeadband 0 for all other nodes.

Specific changes:

1. Added `DataChangeFilter`, `DataChangeTrigger`, `DeadbandType`, `ExtensionObject`, `ObjectId`
   to the top-level `use opcua::types` import block.

2. Modified `harvest_analog_metadata` return type from `()` to `HashMap<NodeId, (f64, f64)>`.
   - Changed spawn_blocking closure to return `(node_id.clone(), *point_id, meta)` instead of
     `(*point_id, meta)` so the NodeId is preserved.
   - Added `eu_ranges: HashMap<NodeId, (f64, f64)>` accumulator that collects nodes with valid
     EU ranges (where `(high - low).abs() > f64::EPSILON`).
   - Returns `eu_ranges` at the end of the function.

3. Updated the call site of `harvest_analog_metadata` (line ~337) to capture the returned map:
   `let eu_ranges = harvest_analog_metadata(...).await;`

4. Added `eu_ranges: &HashMap<NodeId, (f64, f64)>` parameter to `create_subscriptions`.

5. Updated the call site of `create_subscriptions` (line ~412) to pass `&eu_ranges`.

6. Inside `create_subscriptions`, added two closures:
   - `make_percent_filter()` — returns `ExtensionObject` encoding `DataChangeFilter` with
     `DeadbandType::Percent as u32` / `deadband_value: 1.0` / `trigger: StatusValue`.
   - `make_absolute_filter()` — returns `ExtensionObject` encoding `DataChangeFilter` with
     `DeadbandType::None as u32` / `deadband_value: 0.0`.
   Each node gets `make_percent_filter()` if present in `eu_ranges`, else `make_absolute_filter()`.

7. Added `BadDeadbandFilterInvalid` fallback: after `create_monitored_items`, any result with
   `status_code == StatusCode::BadDeadbandFilterInvalid` is collected, logged at warn level, and
   retried with `make_absolute_filter()`. The fallback results patch the original results in-place.
