## What Was Attempted

Added `resolve.dedupe` and `optimizeDeps.include` to `frontend/vite.config.ts` to prevent duplicate React instances. Also added `@radix-ui/react-context-menu` to the `vendor-radix` manual chunk since it was used in the app but missing from the chunk definition.

Root cause: In pnpm's isolated node_modules structure, symlinked packages can resolve React through different virtual-store paths. Without `resolve.dedupe`, Vite may treat these as separate modules, causing the classic "Invalid hook call" error. The UAT failure showed `chunk-AUT5C5CZ.js` with its own React copy — this occurred when the build ran in a worktree context where module paths resolved differently.

## Files Modified

- `frontend/vite.config.ts`: Added `resolve.dedupe: ['react', 'react-dom']` to force single React resolution; added `optimizeDeps.include: ['react', 'react-dom', 'react/jsx-runtime']` to pre-bundle React explicitly in dev server; added `@radix-ui/react-context-menu` to `vendor-radix` manualChunks entry.
