## What Was Attempted

Analyzed the log module's index.tsx. The `ExportButton` was already imported and defined (lines 6, 303-340), but it was only rendered inside the `CompletedTable` component, behind two conditions: `hasExport && instances.length > 0`. This meant the button was invisible when the completed tab had no data (as in UAT), and entirely absent from the Active Logs and Templates tabs.

The fix was to:
1. Move the `ExportButton` into the main page header toolbar (the `div` holding the title "Log" and search form), so it is always visible for any tab when the user has `log:export` permission.
2. Remove the redundant `ExportButton` from inside `CompletedTable` and simplify `CompletedTable`'s props (removed `hasExport` parameter).
3. Remove the now-unused `hasExport` local variable from `LogPage` (the permission check is handled internally by `ExportButton`).

The ExportButton shows row counts contextually: uses `completedData.length` when on the Completed tab, `activeData.length` otherwise.
