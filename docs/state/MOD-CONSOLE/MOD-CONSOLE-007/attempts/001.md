## What Was Attempted

Implemented all four capabilities on PointDetailPanel as specified:

1. **Resize** — Added `resize: 'both'` CSS on the panel div (disabled when minimized) plus a ResizeObserver to capture the new size and persist it to sessionStorage after user drags a resize handle.

2. **Session persistence** — Replaced the bare `useState` position with a `PanelState` object `{ top, left, width, height, minimized }` loaded from `sessionStorage.getItem('io-point-detail-{pointId}')` on first render and written back on every change.

3. **Minimize button** — Added an IconButton in the header that toggles `minimized`. When minimized, the body div is hidden and the panel collapses to its title bar. `resize` is also disabled when minimized.

4. **Pin button** — Added an IconButton in the header. Clicking "Pin" calls `pinPanel()` on the new `pointDetailStore` Zustand store and calls `onClose()` to unmount the local instance. The shell-level `PinnedPointDetailPanels` component in `App.tsx` renders pinned panels outside the route outlet so they survive navigation. Clicking "Unpin" calls `unpinPanel()` + `onClose()`.

5. **Single /detail endpoint** — Primary query now uses `pointsApi.getDetail(pointId)` hitting `GET /api/v1/points/:id/detail`. If that query errors (404 — endpoint not yet implemented), fallback queries using `getMeta` and `getLatest` activate automatically.

6. **New store** — Created `frontend/src/store/pointDetailStore.ts` (Zustand) holding `pinnedPanels[]`. The list is persisted to `sessionStorage` under key `io-point-detail-pinned` so pinned panels are restored after a page refresh.

7. **New API type** — Added `PointDetailResponse` interface and `getDetail()` function to `frontend/src/api/points.ts`.
