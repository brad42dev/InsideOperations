## What Was Attempted

Rewrote the `CornerTrigger` component in `frontend/src/shared/layout/AppShell.tsx` to use a global `document.addEventListener('mousemove', ...)` handler instead of React's `onMouseEnter`/`onMouseLeave` on a transparent div.

The old approach relied on the browser dispatching `mouseenter` on a specific fixed-position div element. This is unreliable in automated testing (Playwright dispatches `mousemove` events at the document level) and can also fail if other elements stack on top of the transparent hit area.

The new approach registers a `mousemove` listener on the document itself, computes whether the cursor is within 48px of each respective corner using raw `clientX`/`clientY` coordinates, and starts/cancels a 1500ms dwell timer. When the timer fires, `showButton` is set to true and the "Exit Kiosk" button appears.

Also added `data-testid="kiosk-corner-trigger-{corner}"` on each corner container and `data-testid="kiosk-exit-button"` on the exit button, so UAT tests can find these elements.

## Why This Attempt Succeeded

The `CornerTrigger` component was already rendering in the DOM when `isKiosk` is true (this was implemented in an earlier commit for MOD-CONSOLE-011). However, the UAT failed because:
1. The component relied on `onMouseEnter` on a transparent fixed div, which Playwright's mouse simulation may not trigger when moving to exact corner coordinates
2. No `data-testid` attributes existed so automated tests couldn't find the elements

The fix uses `document.addEventListener('mousemove')` which fires reliably for all mouse movement including Playwright synthetic events. The coordinate-based detection (`clientX/clientY <= 48`) works regardless of element stacking.
