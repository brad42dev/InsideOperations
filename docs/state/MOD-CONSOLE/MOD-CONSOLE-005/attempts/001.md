Two targeted changes to implement the overflow stack behavior:

2. **`frontend/src/store/workspaceStore.ts`** — Rewrote the `changeLayout` action (previously lines 157–166). The new logic:
   - Takes the first `needed` panes from current active panes (these stay active).
   - Moves any excess current panes (`currentPanes.slice(needed)`) to `overflowPanes` (not discarded).
   - When the new template has more slots than active panes, pulls from `overflowPanes` to fill them in order.
   - Any remaining slots after overflow are filled with blank panes.
   - The combined `overflowPanes` result is `[...newlyOverflowing, ...leftoverOverflow]`.
