## What Was Attempted

Identified and fixed two layered bugs that together caused the TypeError crash:

**Bug 1 (root cause) — API response shape mismatch in `consoleApi.listWorkspaces()`:**
The backend returns `PagedResponse<WorkspaceSummary>` (shape: `{ success, data: WorkspaceSummary[], pagination: {...} }`).
The API client (`client.ts`) detects the `pagination` field and returns `{ data: WorkspaceSummary[], pagination: {...} }` as `T`.
But `consoleApi.listWorkspaces()` was typed as `api.get<WorkspaceLayout[]>(...)`, so `result.data` was cast as `WorkspaceLayout[]` when it was actually an object `{ data: [], pagination: {} }`.
This object was then passed to `setWorkspaces()`, making `workspaces` an object-not-array in the Zustand store.
All subsequent `workspaces.find()` / `workspaces.length` calls either returned `undefined` or threw, which the error boundary caught.

**Bug 2 — Backend response missing `panes` field:**
The backend `WorkspaceSummary` struct only has `{ id, name, metadata, created_at }`. The `panes`, `layout`, and `gridItems` fields are stored inside the `metadata` JSON column. The frontend was never extracting them — it was casting `WorkspaceSummary` as `WorkspaceLayout`, leaving `panes` as `undefined`. This caused `activeWorkspace.panes.reduce(...)` to throw with "Cannot read properties of undefined (reading 'reduce')".

**Fixes applied:**

1. `frontend/src/api/console.ts` — Added `WorkspaceSummary` interface and `normalizeWorkspace()` function that extracts `layout`/`panes`/`gridItems`/`overflowPanes`/`published` from the `metadata` blob and defaults `panes` to `[]`. Fixed `listWorkspaces()` to use `PaginatedResult<WorkspaceSummary>` and extract `.data` array. Fixed `getWorkspace`, `saveWorkspace`, `publishWorkspace` to normalize responses. Fixed `saveWorkspace` to wrap fields in a `metadata` blob matching the backend schema.

2. `frontend/src/pages/console/index.tsx` — Added `(activeWorkspace.panes ?? [])` defensive guard on both `.reduce()` calls (lines 1216/1221) inside the ExportDialog props.

3. `frontend/src/pages/console/WorkspaceGrid.tsx` — Added `safePanes = workspace.panes ?? []` to guard both `presetToGridItems(workspace.layout, safePanes)` and `workspace.panes.map(...)` in the `paneById` useMemo.
