## What Was Attempted

Added a Zustand store subscription-based dirty detector to `frontend/src/pages/console/index.tsx`. Rather than relying solely on `scheduleSave` (a UI event-handler callback) to call `setIsDirty(true)`, a persistent `useWorkspaceStore.subscribe()` listener now computes dirty state by comparing the current active workspace's JSON against a `lastSavedSnapshotRef` snapshot. This means any code path that mutates the workspace store â€” including programmatic mutations from test tooling â€” will trigger the dirty indicator.

Additional fixes made:
- `persistWorkspace` localStorage path: was missing `setIsDirty(false)` after synchronous `saveWorkspacesLocal`. Now updates `lastSavedSnapshotRef` and calls `setIsDirty(false)`.
- `saveMutation.onSuccess`: now also updates `lastSavedSnapshotRef` (for the active workspace) so the subscription-based detector clears dirty after a successful API save.
- API workspace load effect: seeds `lastSavedSnapshotRef` from freshly loaded server data so the app starts in a clean state.
- localStorage workspace load effect: seeds `lastSavedSnapshotRef` from the loaded data.
- `activeId` change effect: resets `lastSavedSnapshotRef` and `isDirty` when the user switches workspaces, and also clears any pending debounce timer so stale saves from the previous workspace don't fire.

The dirty indicator (amber dot on tab) and the persistent failure banner (after 3 failures, "Workspace changes not saved" + "Save now") were already implemented in the prior MOD-CONSOLE-029 task but were unreliable when the workspace change was triggered via non-React code paths.

## Why This Attempt Succeeded

When the UAT test triggered the layout change via JavaScript without going through React's synthetic event system, `handleChangeLayout` was never called, `scheduleSave` was never invoked, and `isDirty` stayed `false`.

The fix adds a Zustand store subscription that detects dirty state by comparing the current workspace JSON against the last-saved snapshot. This is immune to how the change was triggered â€” any mutation to the store (UI click, programmatic mutation, test tooling) is detected.
