## What Was Attempted

Full implementation of the ACME cert renewal systemd timer and API Gateway renewal endpoint:

1. Created `systemd/io-cert-renew.timer` with `OnCalendar=*-*-* 00,12:00:00`, `RandomizedDelaySec=3600`, `Persistent=true`.
2. Created `systemd/io-cert-renew.service` as `Type=oneshot` calling `POST http://127.0.0.1:3000/api/internal/certs/renew` then reloading nginx.
3. Added `rcgen = { version = "0.13", features = ["x509-parser"] }` to workspace `Cargo.toml` and `rcgen.workspace = true` to `services/api-gateway/Cargo.toml`.
4. Added `cert_renew_days: i64` field to `Config` struct in `config.rs` (reads `IO_CERT_RENEW_DAYS`, default 30).
5. Created `services/api-gateway/src/tls.rs` with `ensure_active_cert()` — checks for `active.crt`, generates a self-signed cert via `rcgen::CertificateParams::new()` + `KeyPair::generate()`, writes `self-signed.crt`/`self-signed.key`, symlinks as `active.crt`/`active.key`.
6. Called `tls::ensure_active_cert(&cfg.cert_dir)` from `main.rs` on startup, before the HTTP server starts.
7. Added `renew_cert` handler to `handlers/certificates.rs` — reads `active.crt`, parses expiry via `x509-parser`, checks days vs `IO_CERT_RENEW_DAYS`, returns `{"renewed": false, "days_remaining": N}` (ACME renewal stubbed).
8. Added `POST /api/internal/certs/renew` route to the API router in `main.rs`.
9. Whitelisted `/api/internal/certs/renew` in `mw::is_public_path` so the JWT middleware skips auth for this endpoint.
10. Added `systemctl enable/start io-cert-renew.timer` to `installer/deploy.sh`.
