## What Was Attempted

Modified `frontend/src/pages/dashboards/widgets/ProductionStatusWidget.tsx` to guard against non-array API responses. The original code used `sourcesQuery.data ?? []` and `alarmsQuery.data ?? []`, which only guards against null/undefined. When the API returns a paginated envelope like `{ data: [...], pagination: {...} }`, the client's `request()` function wraps it as a `PaginatedResult<T>` object — which is truthy, bypassing the `??` fallback, and making `.filter()` fail with "sources.filter is not a function".

Fix applied: replaced `?? []` with `Array.isArray(x) ? x : []` for both `sources` and `alarms`. Also added a missing `isError` state that renders "Failed to load status data" instead of silently passing empty arrays.
