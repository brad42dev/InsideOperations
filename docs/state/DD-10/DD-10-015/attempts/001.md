## What Was Attempted

Created 11 new widget component files in `frontend/src/pages/dashboards/widgets/` and registered them all in the `WidgetContainer.tsx` switch statement:

1. **QualityDistributionWidget.tsx** — Fetches OPC point quality counts from `/api/opc/points/current-quality`, falls back to OPC source stats to show a good-quality percentage. Renders a donut pie chart using EChart.
2. **StalePointsWidget.tsx** — Fetches stale points from `/api/opc/points/stale?threshold_minutes=N`. Shows a count KPI + scrollable list. Falls back gracefully to an informative empty state.
3. **BadQualityBySourceWidget.tsx** — Fetches OPC source stats and shows a table with per-source point count and bad-quality count.
4. **PointStatusTableWidget.tsx** — Fetches `/api/opc/points/status` with optional area/filter query params. Shows a table of point name, value, quality, area, and last-update age. Falls back to the points list endpoint.
5. **AlarmHealthKpiWidget.tsx** — Derives a 0–100 composite alarm health score from active alarm count, unacknowledged ratio, and critical alarm count. Shows score, label (HEALTHY/FAIR/POOR/CRITICAL), and stat breakdown.
6. **ProductionStatusWidget.tsx** — Combines OPC source stats and active alarms to derive a NORMAL/DEGRADED/UPSET/OFFLINE status badge with supporting details.
7. **RoundsCompletionWidget.tsx** — Fetches round instances in a configurable time window and shows completion percentage with done/active/missed counts.
8. **OpenAlertsWidget.tsx** — Fetches active human-initiated notifications from `/api/notifications/active`. Shows count KPI plus a list of open alerts with severity color coding.
9. **SystemUptimeWidget.tsx** — Fetches service health and system version. Shows uptime percentage derived from healthy/degraded/unhealthy service ratio, with version and service breakdown.
10. **AlarmRateTrendWidget.tsx** — Similar to AlarmRateWidget but designed for longer windows (default 7 days / 168h). Auto-selects bucket size (24h for >=72h windows, 6h otherwise). Shows alarms/day average.
11. **TrendChartWidget.tsx** — Fetches historical data for a list of point IDs via `pointsApi.historyBatch`. Renders a multi-series time-series line chart using EChart. Supports `points` array config or `variable` binding.

Modified `WidgetContainer.tsx` to import all 11 new components and add case entries for their type strings.
