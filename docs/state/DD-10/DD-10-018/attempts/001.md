## What Was Attempted

Fixed the `results.some is not a function` crash in `TrendChartWidget.tsx`. The widget was using `query.data ?? []` which only guards against `null`/`undefined`, but when the API returns a non-array value (e.g., a paginated response object or unexpected shape), calling `.some()` on it throws a TypeError. The fix replaces the assignment with `Array.isArray(query.data) ? query.data : []`, ensuring `results` is always an array regardless of what the API returns.
