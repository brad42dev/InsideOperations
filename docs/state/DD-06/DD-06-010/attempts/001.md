## What Was Attempted

Four infrastructure gaps in the frontend shell were implemented:

1. **Sidebar state persistence** — Changed `AppShell.tsx` sidebarState `useState` from a static default to a lazy initializer that reads from `localStorage` key `io_sidebar_state`. Added a `useEffect` that writes back on every change, skipping the write when kiosk mode is active (kiosk forces 'hidden' transiently and should not persist that as the user's preference).

2. **BroadcastChannel sync** — Created `frontend/src/lib/broadcastSync.ts` with channel name `io-app-sync`. Exported: `publishThemeChange`, `publishDensityChange`, `publishAuthRefresh`, `publishSessionLock`, `publishSessionUnlock`, `subscribeToSync`. Called publish functions from `store/ui.ts` (`setTheme` → `publishThemeChange`, `lock` → `publishSessionLock`, `unlock` → `publishSessionUnlock`) and `store/auth.ts` (`setAccessToken` → `publishAuthRefresh`). Added `BroadcastSyncReceiver` component in `App.tsx` that subscribes and applies incoming changes using `*Local` store actions to avoid re-broadcasting.

3. **Lazy loading** — Converted all module-level page component imports in `App.tsx` to use `React.lazy()`. Added `<Suspense fallback={<RouteLoadingFallback />}>` wrapping the entire `<Routes>` tree. Created `RouteLoadingFallback` as a minimal inline loading indicator.

4. **Detached window routes** — Added three routes outside the AppShell layout in `App.tsx`:
   - `/detached/console/:workspaceId` → `<WorkspaceView detached />` guarded by `console:read`
   - `/detached/process/:viewId` → `<ProcessView detached />` guarded by `process:read`
   - `/detached/dashboard/:dashboardId` → `<DashboardViewer kiosk />` guarded by `dashboards:read`

   Also added `detached?: boolean` props to `WorkspaceView` and `ProcessView` stub components to satisfy TypeScript.
