## What Was Attempted

Applied the `navigateRef` pattern to the G-key keyboard handler in `/home/io/io-dev/io/frontend/src/shared/layout/AppShell.tsx`.

Specifically:
1. Added `const navigateRef = useRef(navigate)` and `navigateRef.current = navigate` immediately after `const navigate = useNavigate()` at the top of the AppShell component body.
2. Changed `navigate(path)` to `navigateRef.current(path)` in the G-key second-key handler inside the `useEffect`.
3. Changed the `useEffect` dependency array from `[navigate]` to `[]` with an eslint-disable comment, so the listener is registered exactly once on mount and never re-registered.

Root cause: The `useEffect([navigate])` dep array caused the keyboard listener to be torn down and re-registered whenever React processed the state update from `setGKeyHintVisible(true)` (triggered by the G key press). Although React Router's `navigate` is supposed to be stable, using `[]` with a ref ensures the listener is never removed between the G key and the second key press, and ensures `navigate` is always the latest version.
