## What Was Attempted

Added manual lock screen capability to the app shell. The `LockOverlay.tsx` and auth API were
already fully implemented, but there was no way for users to manually trigger the lock. Added:

1. `lockImmediate` boolean field to `ui.ts` store — when true, `LockOverlay` displays immediately
   (for deliberate manual lock actions) rather than waiting for the next interaction.
2. `clearLockImmediate()` action to reset the flag after the overlay acknowledges it.
3. Updated `lock()` action signature to accept `immediate?: boolean` parameter.
4. `LockOverlay.tsx` — reads `lockImmediate` from store, shows overlay immediately when true,
   calls `clearLockImmediate()` after acknowledging. Added `dismissOverlay` and `clearLockImmediate`
   to the useEffect dependency array.
5. `AppShell.tsx` — added `Lock` icon import from lucide-react, added "Lock Screen" button to
   the user menu (above the Sign Out divider), added `Ctrl+L` keyboard shortcut to lock screen
   immediately.
