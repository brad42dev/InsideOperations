## What Was Attempted

Full rewrite of `LockOverlay.tsx` and surrounding lock infrastructure per the
visual-lock-overhaul decision spec.

**LockOverlay.tsx — full rewrite:**
- Passive locked state: overlay hidden until click/keypress (not mousemove)
- Triggering click consumed via capture-phase `mousedown` listener with `stopPropagation()`
- Overlay fades in 300ms / out 200ms via CSS keyframe animation
- `z-index: var(--io-z-visual-lock)` (= 500)
- Lock card varies by `authProvider` + `hasPin`:
  - local + no PIN → password field
  - local + PIN → PIN field + "Use password instead" after 3 failures
  - SSO + no PIN → SSO button
  - SSO + PIN → PIN field + SSO button
- 60-second auto-dismiss (returns to passive locked state, does not unlock)
- 429 rate-limit: countdown timer, input disabled
- 401 forced_signout: brief message then `logout()` + redirect to `/login`
- SSO popup: `window.open()` → postMessage → `verifySsoUnlock` → unlock
- Popup blocked: inline notice in card with sign-out fallback
- Escape in kiosk: dismisses overlay (capture phase, `stopPropagation`) — no unlock
- Sign-out fallback button always visible

**AppShell.tsx changes:**
- Added `wsManager` import
- Added `setLockMeta`, `unlock` to `useUiStore` destructure
- Added `isLockedRef` for Escape guard
- Escape key handler checks `!isLockedRef.current` before exiting kiosk
- Added `idleTimeoutRef` for per-session idle timeout (default `IDLE_TIMEOUT_MS`)
- `resetIdleTimer` uses `idleTimeoutRef.current` instead of hard-coded constant
- Added `bootSyncLock` effect: calls `authApi.sessionCheck()` on mount, reads
  `is_locked`, `auth_provider`, `has_pin`, `idle_timeout_ms` from response.
  If `is_locked = true`: calls `lock()` with meta. Does NOT show overlay.
- Added `wsManager.onSessionLock` / `onSessionUnlock` listeners.
  Note: wsWorker already handles the store update globally; the AppShell listener
  additionally resets the idle timer on unlock for cross-tab sync.

**frontend/src/api/auth.ts:**
- Added `SessionCheckResponse` interface (includes `is_locked`, `auth_provider`,
  `has_pin`, `idle_timeout_ms`, `auth_provider_name`)
- Added `verifyPin(pin)` — POST /api/auth/verify-pin
- Added `verifySsoUnlock(token)` — POST /api/auth/verify-sso-unlock
- Added `sessionCheck()` — GET /api/auth/me (returns SessionCheckResponse)
- Kept `lockSession()`, `verifyPassword()` as-is

**frontend/src/store/ui.ts:**
- Added `LockMeta` interface: `authProvider`, `authProviderName`, `hasPin`
- Added `lockMeta` field to `UiState` (with `DEFAULT_LOCK_META`)
- `lock(meta?)` now accepts optional `Partial<LockMeta>` and merges into state
- Added `setLockMeta(meta)` action for partial updates
- `unlock()` resets `lockMeta` to defaults
