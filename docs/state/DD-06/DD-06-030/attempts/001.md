## What Was Attempted

Added browser fullscreen support to kiosk mode in `frontend/src/shared/layout/AppShell.tsx`:

1. Added `isBrowserFullscreenRef` (useRef) and `needsFullscreenPrompt` (useState) state near kiosk state declarations.
2. Added `fullscreenchange` event listener useEffect that updates `isBrowserFullscreenRef` and auto-dismisses the prompt when fullscreen activates.
3. Modified `enterKiosk()` to call `document.documentElement.requestFullscreen()` and set `needsFullscreenPrompt(true)` in the `.catch()` handler.
4. Updated the URL-param mount useEffect to also call `requestFullscreen()` with the same catch fallback.
5. Modified `exitKiosk()` to call `document.exitFullscreen()` guarded by `document.fullscreenElement` check, and clear the prompt state.
6. Updated the Escape key handler to implement the two-step sequence: if `document.fullscreenElement` is non-null, call `exitFullscreen()`; otherwise call `exitKiosk()`.
7. Added a fullscreen prompt JSX banner (`position: fixed`, top of viewport, zIndex 9000) shown when `isKiosk && needsFullscreenPrompt`, with "Enter fullscreen" and "Skip" buttons.
