## What Was Attempted

Moved `gKeyPending` and `gKeyTimerRef` from `useRef` hooks inside AppShell component to
module-level plain objects (`_gKeyPending`, `_gKeyTimerRef`) so they are not reset when
React Strict Mode double-mounts the component.

Added a module-level `_setGKeyHintVisible` ref that stores the `setGKeyHintVisible` setter
from the live component instance. The component registers its setter in a `useEffect` with
a guard against out-of-order cleanup. All `setGKeyHintVisible(...)` calls inside
`handleKeyDown` were replaced with `_setGKeyHintVisible.current?.(...)` so the keyboard
handler (created once on first mount, due to `[]` deps) always calls the currently-live
setter rather than the orphaned one from the first Strict Mode mount cycle.
