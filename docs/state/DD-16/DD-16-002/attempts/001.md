## What Was Attempted

Implemented adaptive throttling for the data-broker service in 6 steps:

1. Created `services/data-broker/src/throttle.rs` with a `ThrottleLevel` enum (5 levels: Normal, Batch, Deduplicate, ReduceFrequency, OffScreen) and a `compute_throttle()` function that escalates/de-escalates based on FPS and pending-update metrics.

2. Added 7 configurable throttle thresholds to `services/data-broker/src/config.rs`: `throttle_fps_low`, `throttle_fps_high`, `throttle_pending_low`, `throttle_pending_high`, `throttle_global_ratio`, `throttle_global_batch_window_ms`, `throttle_global_deadband` — all with env-var overrides and sensible defaults.

3. Added `throttle_states: Arc<DashMap<ClientId, ThrottleLevel>>` and `global_throttle_active: Arc<AtomicBool>` to `AppState` in `state.rs`.

4. Updated the `StatusReport` handler in `ws.rs` to: read the client's current throttle level, call `compute_throttle()`, store the new level in `throttle_states`, then count all non-Normal clients and compare to total connections — if ratio > `throttle_global_ratio` (default 0.30) the `global_throttle_active` AtomicBool is toggled. Also added `throttle_states.remove()` in the disconnect cleanup.

5. Updated `fanout_batch()` in `fanout.rs` to accept `throttle_states: &DashMap<ClientId, ThrottleLevel>`. For clients at `Deduplicate` or above, accumulation uses point-level deduplication (replacing existing pending entries for a given point_id with the newest value). All existing tests updated to pass an empty `make_throttle_states()`.

6. Updated `run_fanout_flusher()` in `fanout.rs` to accept `throttle_states`, `global_throttle_active`, and `global_deadband`. Per-client skip-counting implements the tiered batch windows: Batch = skip 1 tick (effective 500ms at 250ms base), ReduceFrequency/OffScreen = skip 3 ticks (effective 1s). When global throttle is active and `global_deadband > 0`, points whose absolute value is below the deadband are dropped from pending before send.

7. Wired up in `main.rs`: added `throttle_states` and `global_throttle_active` Arc allocations, passed them to `AppState`, `run_fanout_flusher`, `uds::run_uds_server`, and `notify::run_notify_listener`. Updated `uds.rs` and `notify.rs` function signatures and call sites accordingly.

8. Registered the `throttle` module in `main.rs`.
