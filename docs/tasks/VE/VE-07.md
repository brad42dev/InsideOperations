## Files to Modify

### `frontend/src/shared/hooks/useWsWorker.ts` (around line 264)

Find the `ExportComplete` message handler. Currently it shows a single "report ready" toast. Replace with branching on `kind` and `status`:

```typescript
case 'ExportComplete': {
  const { job_id, kind, status } = msg;
  if (kind === 'video_export') {
    if (status === 'completed') {
      toast.success('Your video is ready', {
        description: 'Click to download',
        action: {
          label: 'Download',
          onClick: () => {
            window.location.href = videoExportsApi.getDownloadUrl(job_id);
          },
        },
        duration: 10_000,
      });
    } else {
      toast.error('Video export failed', {
        description: msg.error_message ?? 'An error occurred during rendering',
        duration: 8_000,
      });
    }
  } else {
    // existing data export toast behaviour — keep unchanged
  }
  break;
}
```

Adjust to match whatever toast library the codebase uses (check existing toast calls elsewhere in the file for the API shape).

If `error_message` is not in the WS message, fetch it: `const job = await videoExportsApi.get(job_id); toast.error('...', { description: job.error_message })`. Only do this if it's cheap (one fetch); otherwise just show a generic message.

### `frontend/src/pages/settings/MyExports.tsx`

A settings sub-page showing both data export jobs and video export jobs in a unified list.

**Data sources:**
- `GET /api/exports` — existing data exports (check `frontend/src/api/exports.ts` for the existing client)
- `GET /api/video-exports` — video export jobs (`videoExportsApi.list()`)

Poll both every 5 seconds when any job is `queued` or `processing` (stop polling when all are terminal). Use `useQuery` with `refetchInterval`.

**Table columns:**
| Column | Notes |
|---|---|
| Type | "Data Export" / "Video Export" |
| Created | relative time (e.g. "2 hours ago") |
| Status | colored badge: queued/processing/completed/failed/cancelled |
| Progress | for video exports in `processing`: `{frames_rendered} / {frames_total} frames` |
| Size | `file_size_bytes` formatted (MB/GB) |
| Actions | Download (completed only), Cancel (queued/processing only), — |

Sort by `created_at DESC`.

**Download action:** `window.location.href = videoExportsApi.getDownloadUrl(job.id)` — triggers browser download.

**Cancel action:** `videoExportsApi.cancel(job.id)` then refetch.

Add a sidebar navigation entry in Settings → "My Exports" (find where other settings nav entries are registered and follow the same pattern).
