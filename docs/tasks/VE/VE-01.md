## What to Do

Add two new permissions to the database and update the design doc. No frontend or backend service code in this phase.

## Files to Create

### `migrations/20260428000001_add_video_export_permissions.up.sql`

```sql
INSERT INTO permissions (name, description, module) VALUES
  ('process:video_export', 'Trigger video exports of historical process playback', 'process'),
  ('console:video_export', 'Trigger video exports of historical console playback', 'console')
ON CONFLICT (name) DO NOTHING;
```

Then grant to roles. First check `migrations/20260314000033_seed_tier1.up.sql` to see whether Admin uses `*` or explicit grants. Follow whatever pattern it uses for the Admin row.

For Shift Supervisor, Operator, and Analyst — find their role IDs and add rows to `role_permissions` using this pattern (check existing grant migrations for the exact SQL shape):

```sql
INSERT INTO role_permissions (role_id, permission_id)
SELECT r.id, p.id
FROM roles r, permissions p
WHERE r.name IN ('shift_supervisor', 'operator', 'analyst')
  AND p.name IN ('process:video_export', 'console:video_export')
ON CONFLICT DO NOTHING;
```

Content Manager, Maintenance, Viewer, Contractor: no grants.

### `migrations/20260428000001_add_video_export_permissions.down.sql`

```sql
DELETE FROM role_permissions
WHERE permission_id IN (
  SELECT id FROM permissions
  WHERE name IN ('process:video_export', 'console:video_export')
);
DELETE FROM permissions
WHERE name IN ('process:video_export', 'console:video_export');
```

- **Build flag required for any cargo build on this machine:** `BINDGEN_EXTRA_CLANG_ARGS="-I/usr/lib/gcc/x86_64-linux-gnu/13/include"` — needed because `auth-service` depends on `samael`. If you run `sqlx migrate run` only (no cargo build), this isn't needed.
- Role name strings in the DB: verify the exact `name` values in the `roles` table before writing the grant SQL (`SELECT name FROM roles;`).
- Admin role: if it has a `permissions = '*'` wildcard grant (check the seed migration), do not add explicit grants for Admin — the wildcard already covers it.
