## What to Do

Two things in one phase:
1. A minimal chrome-free frontend route `/export-render` that renders just the graphic at a given timestamp ã used by the Playwright capture worker
2. A `<TimestampOverlay>` component and CC toggle wired into the live playback bar

These can be done in parallel with VE-01 and VE-02 (frontend-only changes).

## Context

- Rendering is SVG-based via `SceneRenderer` (`frontend/src/shared/graphics/SceneRenderer.tsx`)
- Playback state lives in `frontend/src/store/playback.ts` (Zustand): `mode`, `timestamp`, `timeRange`, `isPlaying`, `speed`
- The playback bar is `frontend/src/shared/components/HistoricalPlaybackBar.tsx`
- Process page: `frontend/src/pages/process/index.tsx` ã uses `<SceneRenderer>` inside an SVG at line ~2029
- Console page: `frontend/src/pages/console/index.tsx`
- App routing: `frontend/src/App.tsx`

## Files to Create

### `frontend/src/pages/export-render/ExportRenderPage.tsx`

Reads URL search params:
- `module` ã `"process" | "console"` (required; redirect to `/` if missing)
- `graphicId` ã UUID string (required)
- `timestamp` ã epoch ms as string (required)
- `width` ã px integer (optional, default `window.innerWidth`)
- `height` ã px integer (optional, default `window.innerHeight`)
- `dpr` ã number (optional, default `window.devicePixelRatio`)
- `overlay` ã `"1"` or `"0"` (default `"0"`)
- `theme` ã `"light" | "dark"` (default: read from existing theme store)

On mount:
```ts
usePlaybackStore.setState({
  mode: 'historical',
  timestamp: tsParam,
  timeRange: { start: tsParam - 1000, end: tsParam + 1000 },
  isPlaying: false,
});
```

Sets `document.body.style.cssText = \`width:${width}px;height:${height}px;overflow:hidden\``.

Dispatches to `<ExportRenderProcess>` or `<ExportRenderConsole>` based on `module` param.

Injects a global style tag: `* { animation: none !important; transition: none !important; }` ã prevents screenshots mid-transition.

### `frontend/src/pages/export-render/ExportRenderProcess.tsx`

- Renders the process graphic at full page size with no nav/sidebar/playback bar
- Reuses the same `SceneRenderer` and historical data-fetch hooks that `pages/process/index.tsx` uses
- After all historical point queries settle AND `document.fonts.ready` resolves, sets `document.body.setAttribute('data-export-ready', 'true')`
- If `overlay === "1"`, renders `<TimestampOverlay timestamp={tsParam} />` absolutely positioned bottom-center inside the SVG/graphic container

**Detecting "all queries settled":** Watch `useIsFetching({ queryKey: ['historical'] })` ã when it drops to `0` and has been `0` for one tick (use a `useEffect`), and `document.fonts.ready` is resolved, set the attribute.

### `frontend/src/pages/export-render/ExportRenderConsole.tsx`

Same pattern but for the Console module. Renders the workspace/grid graphic at full page size, no chrome. Uses the same ready-signal logic.

### `frontend/src/shared/components/TimestampOverlay.tsx`

Pure presentational component:

```tsx
interface Props { timestamp: number }

export function TimestampOverlay({ timestamp }: Props) {
  const label = formatExportTimestamp(timestamp); // YYYY-MM-DD HH:mm:ss
  return (
    <div style={{
      position: 'absolute',
      bottom: 24,
      left: '50%',
      transform: 'translateX(-50%)',
      padding: '6px 14px',
      background: 'rgba(0,0,0,0.6)',
      color: 'white',
      font: '600 18px ui-monospace, "SF Mono", "Cascadia Mono", monospace',
      borderRadius: 4,
      pointerEvents: 'none',
      zIndex: 9999,
      whiteSpace: 'nowrap',
    }}>
      {label}
    </div>
  );
}
```

`formatExportTimestamp(ts: number): string` ã format as `YYYY-MM-DD HH:mm:ss` in local time. Do not use `en-CA` locale directly (it can inject a comma); instead pad manually:
```ts
const d = new Date(ts);
const pad = (n: number) => String(n).padStart(2, '0');
return `${d.getFullYear()}-${pad(d.getMonth()+1)}-${pad(d.getDate())} ${pad(d.getHours())}:${pad(d.getMinutes())}:${pad(d.getSeconds())}`;
```

## Files to Modify

### `frontend/src/App.tsx`

Register `/export-render` outside `<AppShell>` so no nav/sidebar renders. Place it next to `/login` in the route tree. It should still be wrapped in the auth provider (so `useAuthStore` works) but NOT inside the kiosk/visual-lock middleware:

```tsx
<Route path="/export-render" element={<ExportRenderPage />} />
```

### `frontend/src/store/playback.ts`

Add two fields:
```ts
ccOverlayEnabled: boolean;
setCcOverlayEnabled: (v: boolean) => void;
```

`setCcOverlayEnabled` persists to `localStorage` with key `io_playback_cc` and updates store state. Initialize from `localStorage.getItem('io_playback_cc') === '1'`.

### `frontend/src/shared/components/HistoricalPlaybackBar.tsx`

1. Read `ccOverlayEnabled` and `setCcOverlayEnabled` from `usePlaybackStore`.
2. Add a CC toggle button in the controls area (between existing playback controls and speed selector). Use text `CC` styled as a pill button ã active: accent color, inactive: muted. Toggle calls `setCcOverlayEnabled(!ccOverlayEnabled)`.
3. When `ccOverlayEnabled` is true, render `<TimestampOverlay timestamp={timestamp} />` using `createPortal(overlay, document.body)` so it sits above the graphic regardless of CSS stacking context. Only render in `mode === 'historical'` and only when the bar is mounted (i.e., don't render in live mode).

## Gotchas

- **`position: fixed` breaks inside react-grid-layout transforms.** Use `createPortal(el, document.body)` for the overlay in the live playback bar. The `ExportRenderPage` doesn't have this constraint (no grid layout), so `position: absolute` is fine there.
- **`waitUntil: 'networkidle'`** in Playwright is unreliable with a persistent WebSocket. The `data-export-ready` attribute is the authoritative signal ã set it last, after queries settled AND fonts ready.
- **RBAC:** The `/export-render` route still needs `process:read` or `console:read` in the auth provider. The export ticket JWT (minted in Phase 5) carries these read-tier permissions. Do not make the route fully public.
