## Hardening Tasks

### 1. Orphan Recovery on Service Startup

In `services/video-export-service/src/main.rs`, after the DB pool is established and before the job queue starts accepting new work, run:

```sql
UPDATE video_export_jobs
SET status = 'failed',
    error_message = 'Service restarted during render',
    updated_at = NOW()
WHERE status IN ('queued', 'processing')
RETURNING id;
```

For each returned id, emit `pg_notify('export_complete', '{"job_id":"...","kind":"video_export","status":"failed"}')` so the user gets a toast rather than silent disappearance.

Also delete any `.tmp` directories under `${EXPORT_DIR}/videos/.tmp/` on startup (partial renders from the crashed process).

### 2. Disk Space Pre-flight

In `services/video-export-service/src/handlers.rs`, before enqueuing a new job, estimate the required disk space and reject if insufficient.

**Estimate formula:**
```
frames = (range_end_ms - range_start_ms) / step_ms
estimated_bytes = frames * width_px * height_px * 4 * 0.15  // 4 bytes/px raw, ~15% VP9 ratio for this content
required_bytes = estimated_bytes * 1.5  // 1.5× headroom
```

Check free space via `statvfs` (use the `nix` crate or call `std::fs::metadata` + platform-specific). If `free < required_bytes`, return:
```json
HTTP 507 { "error": "Insufficient disk space for this export. Try a shorter range or larger step interval." }
```

**Alternative simpler check:** if you don't want to add `nix` as a dependency, use `std::process::Command::new("df").arg("-B1").arg(&export_dir)` and parse the output. Less portable but avoids a new dep.

### 4. File Retention Cleanup

Add a cleanup task to `services/video-export-service/src/main.rs` that runs hourly (use `tokio::time::interval`):

```rust
async fn cleanup_old_exports(pool: &PgPool, cfg: &Config) {
    let cutoff = Utc::now() - Duration::hours(cfg.export_retention_hours as i64);
    
    // Find expired completed/failed jobs with files
    let expired = sqlx::query!(
        "SELECT id, file_path FROM video_export_jobs
         WHERE created_at < $1 AND file_path IS NOT NULL",
        cutoff
    ).fetch_all(pool).await.unwrap_or_default();
    
    for row in expired {
        if let Some(path) = row.file_path {
            let _ = tokio::fs::remove_file(&path).await;
        }
        let _ = sqlx::query!(
            "UPDATE video_export_jobs SET file_path = NULL, status = 'cancelled' WHERE id = $1",
            row.id
        ).execute(pool).await;
    }
    
    // Also sweep the .tmp directory for anything older than 2 hours
    // (partial renders that weren't cleaned up)
}
```
