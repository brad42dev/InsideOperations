## What to Do

Build the new `video-export-service` Rust crate and wire the api-gateway proxy routes. The render worker is a stub that immediately marks the job failed ‚Äî the goal is exercising the full pipeline (gateway ‚Üí service ‚Üí DB ‚Üí pg_notify ‚Üí data-broker ‚Üí WebSocket ‚Üí frontend toast) before any real rendering exists.

Also extend the existing `pg_notify` / `WsServerMessage` infrastructure to carry `kind` and `status` fields so the frontend can distinguish video exports from data exports.

## Context

- Existing export notification pipeline: `services/api-gateway/src/handlers/exports.rs:335` emits `pg_notify('export_complete', '{"job_id":"..."}')` ‚Üí `services/data-broker/src/notify.rs:127` consumes it ‚Üí `WsServerMessage::ExportComplete { job_id }` ‚Üí frontend toast in `frontend/src/shared/hooks/useWsWorker.ts:264`
- Existing export dir: `state.config.export_dir` in `services/api-gateway/src/config.rs`
- Workspace `Cargo.toml` lists all services under `members`
- Port 3011 ‚Äî verify it's unused: `grep -r "3011" services/*/src/`

### `services/video-export-service/src/handlers.rs`

Internal endpoints (called by gateway, not by browsers directly):
- `POST /internal/jobs` ‚Äî deserialize `CreateJobRequest`, insert row, enqueue, return `202 { id, status, frames_total }`
- `DELETE /internal/jobs/:id` ‚Äî cancel in-flight job, update DB
- `GET /internal/jobs/:id` ‚Äî return job row as JSON

### `services/api-gateway/src/handlers/video_exports.rs` (new file)

User-facing gateway handlers (proxied from the browser):

**`create_export`** ‚Äî `POST /api/video-exports`:
- Extract claims, check `<module>:video_export` permission (compute: `format!("{}:video_export", req.module)`)
- Check for existing in-flight job:
  ```sql
  SELECT id FROM video_export_jobs
  WHERE created_by = $1 AND status IN ('queued','processing')
  LIMIT 1
  ```
  If found ‚Üí 409.
- Validate:
  - `range_end > range_start`
  - duration ‚â§ 4 hours
  - `step_ms` in [100, 60_000]
  - `fps` in [1, 30]
  - `width_px * height_px ‚â§ 3840 * 2160`
  - `(range_end_ms - range_start_ms) / step_ms ‚â§ 14_400` (max frames)
- Forward to `POST {VIDEO_EXPORT_SERVICE_URL}/internal/jobs`
- Return 202 with job id

**`list_exports`** ‚Äî `GET /api/video-exports`:
- Query `video_export_jobs WHERE created_by = user_id ORDER BY created_at DESC LIMIT 50`
- Return array

**`get_export`** ‚Äî `GET /api/video-exports/:id`:
- Query single row, enforce `created_by = user_id`

**`cancel_export`** ‚Äî `DELETE /api/video-exports/:id`:
- Enforce ownership, forward to `DELETE {VIDEO_EXPORT_SERVICE_URL}/internal/jobs/:id`

**`download_export`** ‚Äî `GET /api/video-exports/:id/download`:
- Query `file_path`, `original_filename`, `file_size_bytes` from `video_export_jobs`
- Enforce `created_by = user_id` and `status = 'completed'`
- Stream file from disk using `tokio::fs::File` + `tokio_util::io::ReaderStream`
- Headers: `Content-Type: video/webm`, `Content-Disposition: attachment; filename="<original_filename>"`, `Content-Length: <file_size_bytes>`

## Gotchas

- **Verify port 3011 is unused:** `grep -r "3011" services/*/src/ crates/*/src/` before using it.
