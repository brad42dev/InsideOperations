## What This Feature Should Do

The API Gateway should check and renew TLS certificates automatically, triggered by a systemd timer that runs every 12 hours. The timer calls an internal endpoint on the API Gateway (`POST /api/internal/certs/renew`), which checks certificate expiry and initiates ACME renewal if needed. After renewal, nginx is reloaded so the new certificate takes effect without downtime.

The spec also requires the API Gateway to auto-generate a self-signed certificate on first startup (via `rcgen`) so HTTPS works immediately without manual setup.

## Spec Excerpt (verbatim)

> **Auto-renewal**: A systemd timer triggers the API Gateway's renewal check every 12 hours. Certificates are renewed when within 30 days of expiry. Post-renewal hook reloads nginx (zero downtime — SIGHUP to master process, workers drain gracefully).
>
> ```ini
> # /etc/systemd/system/io-cert-renew.timer
> [Timer]
> OnCalendar=*-*-* 00,12:00:00
> RandomizedDelaySec=3600
> Persistent=true
> ```
>
> ```ini

> Type=oneshot

> ExecStartPost=/bin/bash -c 'nginx -t && systemctl reload nginx'
> ```

## Fix Instructions

**Step 1 — Create the systemd units:**

Create `/home/io/io-dev/io/systemd/io-cert-renew.timer`:
```ini
[Unit]
Description=Inside/Operations ACME Certificate Renewal

[Timer]
OnCalendar=*-*-* 00,12:00:00
RandomizedDelaySec=3600
Persistent=true

[Install]
WantedBy=timers.target
```

Create `/home/io/io-dev/io/systemd/io-cert-renew.service`:
```ini
[Unit]
Description=Inside/Operations ACME Certificate Renewal Check

[Service]
Type=oneshot
ExecStart=/usr/bin/curl -sf -X POST http://127.0.0.1:3000/api/internal/certs/renew
ExecStartPost=/bin/bash -c 'nginx -t && systemctl reload nginx'
User=root
```

Note: the spec uses port 3001 (Data Broker) in the example, but the cert renew endpoint belongs on port 3000 (API Gateway). Use 3000.

**Step 2 — Add `rcgen` to API Gateway dependencies:**

In `/home/io/io-dev/io/Cargo.toml` `[workspace.dependencies]`, add:
```toml
rcgen = { version = "0.13", features = ["x509-parser"] }
```

In `/home/io/io-dev/io/services/api-gateway/Cargo.toml`:
```toml
rcgen.workspace = true
```

**Step 3 — Implement self-signed cert auto-generation:**

In `services/api-gateway/src/handlers/certificates.rs` (or a new `src/tls.rs`), add a function that:
1. Checks if `IO_CERT_DIR/active.crt` exists at startup
2. If not, generates a self-signed cert using `rcgen` with the server's hostname as CN and SANs
3. Writes `self-signed.crt` and `self-signed.key` to `IO_CERT_DIR`
4. Creates `active.crt` and `active.key` symlinks pointing to the self-signed files

Call this function from `api-gateway/src/main.rs` before starting the HTTP server.

**Step 4 — Add `POST /api/internal/certs/renew` endpoint:**

Add a handler in `services/api-gateway/src/handlers/certificates.rs` that:
1. Reads the current cert at `IO_CERT_DIR/active.crt`
2. Checks days remaining
3. If within `IO_CERT_RENEW_DAYS` of expiry and the cert is an ACME cert, initiates renewal
4. Returns 200 with `{"renewed": true/false, "days_remaining": N}`

Register the route in `src/main.rs` — it should not require JWT auth (called by the systemd service as root).

**Step 5 — Update `installer/deploy.sh` to enable the timer:**

Add to the "Enable and start services" step:
```bash
systemctl enable io-cert-renew.timer
systemctl start io-cert-renew.timer
```

Do NOT:
- Implement full ACME/Let's Encrypt renewal in the first pass if the scope is too large — the timer and self-signed auto-generation are the MVP; ACME renewal can be stubbed to return `{"renewed": false}` initially
- Use certbot or any external process — all cert operations must be native Rust in the API Gateway
- Put the renew endpoint behind JWT auth — the systemd service calls it as root with no token
