# DD-39-013 — Designer home page crashes with "dashboards.map is not a function" error

**Unit:** DD-39
**Wave:** 3
**Priority:** high
**Source:** UAT
**Status:** pending

---

## Problem

The Designer home page (`/designer`) throws an error boundary with the message
`dashboards.map is not a function` immediately on load.

**Root cause:** `dashboardsApi.list()` is declared in `src/api/dashboards.ts` with the
return type `Promise<ApiResult<Dashboard[]>>`, but the backend `/api/dashboards` endpoint
returns a *paginated* envelope:

```json
{ "success": true, "data": [...], "pagination": { "page": 1, ... } }
```

The API client (`src/api/client.ts`) detects the `pagination` key and wraps the
response as `{ data: Dashboard[], pagination: {...} }` — i.e. a `PaginatedResult<Dashboard>`.
Callers that do `return r.data` therefore receive a plain object, not an array.
The `?? []` fallback never fires (the value is truthy), so `.map()` crashes at runtime.

All four callers of `dashboardsApi.list()` are affected:
1. `src/pages/designer/DesignerHome.tsx`
2. `src/pages/designer/DesignerDashboardsList.tsx`
3. `src/pages/dashboards/PlaylistManager.tsx`
4. `src/pages/dashboards/index.tsx`

---

## Files to Create or Modify

- `frontend/src/api/dashboards.ts` — fix `list()` return type to `PaginatedResult<Dashboard>`
- `frontend/src/pages/designer/DesignerHome.tsx` — use `r.data.data` in queryFn
- `frontend/src/pages/designer/DesignerDashboardsList.tsx` — use `r.data.data` in queryFn
- `frontend/src/pages/dashboards/PlaylistManager.tsx` — use `result.data.data` in queryFn
- `frontend/src/pages/dashboards/index.tsx` — use `result.data.data` in queryFn

## Acceptance Criteria

1. `dashboardsApi.list()` is typed as `Promise<ApiResult<PaginatedResult<Dashboard>>>`.
2. All four callers extract the array via `r.data.data` (not `r.data`).
3. The `?? []` fallbacks in all four components remain valid (defensive).
4. `npx tsc --noEmit` passes with zero errors.
5. The Designer home page loads without an error boundary.
