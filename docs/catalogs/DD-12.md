## Summary

The Forensics module has a substantial working skeleton: investigation lifecycle (create/open/close/cancel), multi-stage evidence model with all 10 evidence types registered, threshold search with list/trend toggle, correlation heatmap via ECharts, and RBAC + error-boundary guards on all routes. Seven significant gaps remain: the alarm search entry point is absent, the threshold search trend view renders a placeholder instead of a real chart, the PointContextMenu signature does not match the CX-POINT-CONTEXT spec (missing `tagName`/`isAlarm`/`isAlarmElement` props and Forensics never wires it up), stage drag-to-reorder is missing, the investigation toolbar lacks Export/Share/Print actions, the left panel shows no data-link context badges or Recent Investigations section, and annotation updates are not persisted to the server.
