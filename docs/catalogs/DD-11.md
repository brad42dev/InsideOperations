## Summary

The Reports module frontend is substantially implemented: template browser, config panel, history tab, schedules tab, ReportViewer with sandboxed iframe, and async generation with polling are all present. However, five gaps are confirmed: (1) PDF generation falls back to HTML because the `typst-as-lib` crate is not wired in; (2) only 20 Phase 1 canned report templates are seeded — 18 Phase 2 templates are missing; (3) the backend RBAC check uses a non-canonical permission name (`reports:run` instead of `reports:create`/`reports:export`); (4) the `export_complete` WebSocket push from the data-broker is not implemented (pg_notify fires but data-broker has no LISTEN on that channel); (5) the "Equipment & Maintenance" category is absent from the frontend CATEGORIES filter array. Four Wave 0 gaps are also recorded.
