## Summary

The Export System spec (doc 25) defines three capabilities: Universal Export (POST /api/exports with 6 formats and a full export dialog), Admin Bulk Update (4-step wizard with XLSX support, column mapping, conflict detection, and 8+ entity types), and Change Snapshots (point-in-time restore with selective row restore and pre-restore snapshots). The Universal Export backend and dialog are entirely absent — no `/api/exports` route exists anywhere in the codebase. The Bulk Update handler covers 4 entity types instead of the 8+ mandated and only accepts CSV (not XLSX). The Snapshot schema and handler depart from the spec-defined schema. The "My Exports" page exists but is scoped to report jobs, not universal export jobs. 7 gaps found: 3 high-priority (missing Universal Export backend, missing Export Dialog UI, bulk update entity coverage), 4 medium-priority (XLSX bulk update, conflict detection, selective restore, permission mismatch).

## Wave 0 Contract Gaps

DD-25 is a Wave 3 unit. The CX-EXPORT contract from Wave 0 is the primary authority for per-module export buttons; DD-25 defines the backend and shared export system that those buttons depend on. The backend gaps here (no `/api/exports` endpoint, no ExportFormatter) are prerequisites for any module satisfying CX-EXPORT.
