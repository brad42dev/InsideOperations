## Summary

The TimescaleDB schema (hypertable, five continuous aggregates, dedup constraint, fillfactor, backfill function) is correctly created in migrations. The archive service's maintenance loop handles compression and raw retention correctly. However six gaps exist: the REST API is missing the 15-minute and 1-day resolution tiers; the maintenance loop omits 15m and 1d from both aggregate retention and refresh; no rolling-average endpoint is implemented; the raw-resolution query incorrectly filters to Good quality (forensic analysis requires all qualities); the `points_history_raw` FK uses `ON DELETE RESTRICT` instead of `ON DELETE CASCADE`; and the compression settings do not explicitly set the Gorilla algorithm.
