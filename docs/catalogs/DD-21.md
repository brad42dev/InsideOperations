## Summary

The API gateway correctly implements the core response envelope (`{ success, data, message? }` via `ApiResponse<T>`), error envelope (`{ success: false, error: { code, message, details? } }` via `IoError`), JWT authentication, and token-bucket rate limiting. Four gaps were found: list endpoints return a flat array instead of the specified paginated envelope; rate-limit headers are absent from non-429 responses; the SCIM token admin path does not match the spec; and the `validator` crate is absent despite the spec requiring it for input validation.
