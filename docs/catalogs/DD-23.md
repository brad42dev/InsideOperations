## Summary

The expression builder component exists and covers the basic tile-palette UI, useReducer state management, drag-and-drop reordering within the workspace, and expression preview text. However, the implementation has critical architectural gaps: the AST serialization format stored in `ExpressionAst` is a flat tile array (`tiles[]`) rather than the spec-defined tree format (`root: ExprNode`), which means the backend Rhai evaluator cannot consume it. The `field_ref` tile type required for rounds and log contexts is entirely absent. Sixteen gaps were identified, eight of which are high priority.
