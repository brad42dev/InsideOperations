## Summary

The IPC contract layer has significant structural gaps. The `io-models` crate is missing
all shared domain types defined in §8 (PointValue, Event, AlertDispatch, UserIdentity,
SourceStatus, Permission enum, WsTicket — none exist). The Rust `WsServerMessage` in
`io-bus` uses an incorrect wire shape for the `Update` variant that disagrees with both
the spec and the TypeScript definition. Five of nine NOTIFY payload types from §5 are
absent from `io-bus`. The API key (`io_sk_` path) in the auth middleware has no
implementation. Six task files generated; one high-priority architectural gap (missing
domain types), two high-priority structural wire format gaps, three medium.
