## Summary

The DD-29 auth-service is substantially implemented: local auth, OIDC, SAML, LDAP, TOTP MFA, SMS MFA, email MFA, recovery codes, SCIM Users endpoints, API keys, EULA endpoints, and WebSocket tickets are all present. However, eight gaps were found ranging from critical security defects to missing features. The most severe are: (1) the EULA gate is not enforced — local login issues tokens even when EULA has not been accepted; (2) OIDC ID token signature is not validated against JWKS — the comment at oidc.rs:295 explicitly notes this is incomplete; (3) SCIM Groups endpoints are entirely absent; (4) MFA policy enforcement (mfa_required gate, grace period, per-role policies) is not implemented in any login flow; and (5) Duo Security MFA is not implemented. The IdP role mapping query references a wrong column name (`idp_group` instead of `idp_group_value` per the schema). Background cleanup tasks are absent. These are medium-to-high severity gaps.
