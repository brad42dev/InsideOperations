## Summary

The email service has a correct structural skeleton — the Axum service, PostgreSQL-backed queue, `lettre` integration, and `minijinja` dependency are all present. However several significant gaps exist: the template engine uses a hand-rolled `{{var}}` regex substitution instead of the specced MiniJinja engine; the queue worker polls every 10s (spec: 1s) with batch size 3 (spec: 10); four of six provider adapters are missing (only SMTP and Webhook implemented, no MS Graph, Gmail, or SES); the suppression list table and pre-send suppression check are entirely absent; nine required API endpoints are missing from the router; the `oauth2` crate required for XOAUTH2/Graph/Gmail token management is not in the workspace; the `test_provider` handler is a stub that unconditionally sets `last_test_ok = true` without sending a real test email; and the config struct is missing the worker-tuning environment variables specified in the deployment section. Wave 0 contracts do not apply to this backend-only service.
