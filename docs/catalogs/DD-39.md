## Summary

The .iographic format infrastructure exists as scaffolding — import and export endpoints compile and run — but the implementation is missing most of the spec's core features. The export produces a structurally incomplete ZIP (manifest lacks `shapes`, `stencils`, `shape_dependencies`, `point_tags`, and `checksum` fields; `graphic.json` writes raw UUID-based bindings instead of portable tag-based ones). The import wizard frontend is fully built but the two backend endpoints it expects (`analyze` and `commit`) do not exist at the paths the frontend calls. The MIME type is wrong. Custom shape sidecars are not exported. Expressions are not serialized. Eight task files were created — three high priority, five medium.
