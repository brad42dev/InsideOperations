## Designer Module — Custom Shapes Management UI

Scenario 1: [DD-39-010] Designer page renders without error — navigate to /designer → page loads with no error boundary ("Something went wrong") visible

Scenario 2: [DD-39-010] Symbol Library accessible from Designer — navigate to /designer, look for a Symbols/Library/Palette panel or button → some form of shape library panel is visible

Scenario 3: [DD-39-010] Custom shapes section visible in Symbol Library — navigate to /designer/symbols or open Symbol Library in Designer → a "Custom", "My Shapes", or "User Shapes" section is visible alongside built-in ISA-101 categories

Scenario 4: [DD-39-010] Custom shapes empty state shown (not hidden) — when no custom shapes exist, navigate to custom shapes section → an empty state message ("No custom shapes", "No custom shapes yet", or similar) is displayed rather than nothing

Scenario 5: [DD-39-010] Symbol library shows ISA-101 built-in categories — navigate to Designer symbol palette → built-in ISA-101 categories (e.g., "Vessels", "Instruments", or similar equipment categories) are visible
